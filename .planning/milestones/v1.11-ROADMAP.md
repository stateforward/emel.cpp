# Roadmap: EMEL

## Current Milestone: v1.11 TE-75M GGUF Trimodal Embedding Runtime

This milestone adds one truthful maintained `augmem/TE-75M-GGUF` embedding slice to EMEL. It is
fixed to `TE-75M-q8_0.gguf`, explicit `omniembed` model-family support, synchronous `text`,
`image`, and `audio` embedding lanes, one shared normalized embedding contract, and proof via
upstream golden baselines plus tiny cross-modal smoke checks. It explicitly excludes generic media
decode, public API commitments, vector search, and broader TE/`omniembed` scope.

## Phase Details

### Phase 47: TE Truth Anchor
**Goal**: Maintain one exact TE fixture and one exact proof corpus before runtime work broadens.
**Depends on**: Phase 46.1
**Requirements**: FIX-01, FIX-02
**Success Criteria** (what must be TRUE):
  1. `tests/models/TE-75M-q8_0.gguf` is the documented maintained TE fixture with source URL,
     license, size, and checksum recorded under `tests/models/README.md`.
  2. The milestone docs and test seams name one exact TE file instead of implying broad TE or
     `omniembed` support.
  3. A tiny canonical text/image/audio proof corpus is defined so later phases can compare against
     the same upstream truth set.
**Plans**: 47-01

### Phase 48: Omniembed Model Contract
**Goal**: Add truthful `omniembed` acceptance and TE execution bindings without aliasing to a
generation-only LLM path.
**Depends on**: Phase 47
**Requirements**: MOD-01, MOD-02
**Success Criteria** (what must be TRUE):
  1. EMEL recognizes `gguf.architecture=omniembed` and validates required TE tensor families for
     text, image, audio, and projection heads explicitly.
  2. Unsupported or malformed TE files fail deterministically instead of slipping through an
     existing LLM architecture path.
  3. The TE execution contract exposes the bindings later modality lanes need without widening
     public API scope.
**Plans**: 48-01

### Phase 49: Text Embedding Lane
**Goal**: Produce maintained TE text embeddings on the shared-space contract.
**Depends on**: Phase 48
**Requirements**: TXT-01, TXT-02
**Success Criteria** (what must be TRUE):
 1. Operator can submit maintained text input and receive a normalized 1280-dimensional TE
     embedding.
 2. Supported Matryoshka truncation to `768`, `512`, `256`, and `128` dimensions works with
     renormalization.
 3. The text lane reuses repo-owned request shaping where appropriate without conflating tokenizer
     encoders with TE model encoders.
**Plans**: 49-01

### Phase 50: Vision Embedding Lane
**Goal**: Add the maintained in-memory image path and TE vision encoder runtime.
**Depends on**: Phase 49
**Requirements**: VIS-01, VIS-02
**Success Criteria** (what must be TRUE):
  1. Operator can submit one documented in-memory image payload and receive a normalized
     TE embedding.
  2. Invalid image payload shapes or preprocessing contracts are rejected explicitly.
  3. The phase stays scoped to inference-time image preprocessing and TE vision encoding, not
     generic image file decode/import.
**Plans**: 50-01

### Phase 51: Audio Embedding Lane
**Goal**: Add the maintained in-memory audio path and TE audio encoder runtime.
**Depends on**: Phase 50
**Requirements**: AUD-01, AUD-02
**Success Criteria** (what must be TRUE):
  1. Operator can submit one documented in-memory audio payload and receive a normalized
     TE embedding.
  2. Invalid audio payload shapes or preprocessing contracts are rejected explicitly.
  3. The phase stays scoped to inference-time audio preprocessing and TE audio encoding, not broad
     decode/resample/transcode support.
**Plans**: 51-01

### Phase 52: Shared Embedding Session
**Goal**: Unify modality outputs behind one deterministic TE embedding contract.
**Depends on**: Phase 51
**Requirements**: EMB-01, EMB-02
**Success Criteria** (what must be TRUE):
  1. Text, image, and audio lanes all return one consistent normalized embedding result contract.
  2. Supported truncation dimensions and unsupported-dimension rejection behave uniformly across
     modalities.
  3. The first maintained TE slice remains synchronous, bounded, and limited to one modality per
     request.
**Plans**: 52-01

### Phase 53: TE Proof And Regression
**Goal**: Make the TE maintained slice provable and regression-resistant before any support claim
widens further.
**Depends on**: Phase 52
**Requirements**: PRF-01, PRF-02
**Success Criteria** (what must be TRUE):
  1. Maintained tests compare EMEL output against stored upstream TE golden embeddings for
     canonical text, image, and audio fixtures.
  2. Tiny cross-modal smoke tests prove the maintained slice lands the canonical triplet set in one
     usable shared space.
  3. The TE proof integrates into the normal repo quality-gate workflow instead of relying on ad
     hoc manual validation.
**Plans**: 53-01
