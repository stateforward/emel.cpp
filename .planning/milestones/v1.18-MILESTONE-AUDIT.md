## Source-Backed Checks

| Claim | Live Source Evidence | Result |
|-------|----------------------|--------|

| Runner owns CLI/config parsing | `tools/paritychecker/parity_runner.cpp` owns usage text, argument parsing, text-file loading, validation, manifest operations, and calls `run_parity(args.parity)` only after validation. | Passed |
| Runner no longer owns bulk mode implementation | `run_parity(...)` resolves `find_engine(opts.mode)` and invokes `engine->run(opts)`; bulk mode code is not in the runner. | Passed |
| Engine registration is explicit and fail-closed | `tools/paritychecker/parity_engine.cpp` maps tokenizer, GBNF, kernel, Jinja, and generation modes to `engine_adapter` records and returns `nullptr` by default. | Passed |
| Build registration is modular | `tools/paritychecker/CMakeLists.txt` defines separate CLI, runner, manifest, engine-registration, engine-implementation, tokenizer-engine, reference-support, and common source groups. | Passed |
| EMEL/reference lanes are isolated | Reference lane owns llama model/context/vocab state; EMEL lane loads and drives EMEL model/generator state separately. Source tests guard shared lane-object regressions. | Passed |
| Generation parity truth is live reference output | `run_generation_harness_contract(...)` calls `run_reference_generate(state.reference, opts, reference_result)` before baseline load. | Paritychecker avoids non-kernel actor internals | Broad source guard scans paritychecker `.cpp`/`.hpp` files for actor action/guard/detail includes and detail namespace reaches, allowing only approved kernel detail arithmetic surfaces. Manual source scan found no forbidden reach. | Passed |
| Dependency manifest covers registered parity modes | `dependency_manifest::records()` contains records for tokenizer, GBNF parser, kernel, Jinja, and generation, including source/config/fixture/model/script/snapshot kinds where applicable. | Passed |
| Dependency manifest has maintained production emission | `paritychecker --write-dependency-manifest <path>` emits `parity_dependency_manifest/v1`; the generated baseline is checked in at `tools/paritychecker/dependency_manifest.txt`. | Passed |
| Manifest freshness forces the parity gate | `paritychecker --check-dependency-manifest <path>` reports `full_gate=1` and exit `3` for missing, stale, or uncertain states. `scripts/quality_gates.sh` runs `scripts/paritychecker.sh` when this happens. | Passed |
| Manifest format is documented | `tools/paritychecker/dependency_manifest.md` documents the CLI emission path, freshness check path, and uncertain override example. | Passed |
