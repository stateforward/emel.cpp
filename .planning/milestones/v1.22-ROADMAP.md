# Roadmap: EMEL v1.22 Weight Loading Ownership Cutover Gap Closure

**Milestone:** v1.22 Weight Loading Ownership Cutover
**Status:** Complete after strict source-backed audit closeout and state metadata repair
**Source:** `.planning/milestones/v1.22-MILESTONE-AUDIT.md`
**Started:** 2026-05-02
**Reopened:** 2026-05-03

## Goal

Close the maintained-path and strict source-backed audit gaps found after v1.22 archival. The
benchmark, parity, and probe model-loading lanes must not allocate GGUF KV arena or entry storage
inside `model_loader.process_event(...)`, the loader/tensor actor code must satisfy the
milestone's SML routing rules in live source, and the closeout state must truthfully name the final
strict source-backed closure phases.

## Scope

This is a focused gap-closure milestone for the v1.22 ownership cutover. It preserves the tensor-owned
loading boundary, keeps concrete I/O strategies deferred, and uses the user's explicit permission
to update snapshots, benchmarks, and models when those updates are required by the maintained-path
fix. Phase 196 is metadata-only; it records and verifies the final Phase 195/196 closeout truth.

Out of scope:
- adding a concrete `emel/io` runtime
- widening model-family support
- replacing the tensor-owned loader architecture with a new owner

## Phases

Historical Phase 185-193 details are preserved in the earlier v1.22 archive history. Phases 194,
195, and 196 are reopened gap-closure phases added after source-backed audits.

### Phase 194: Maintained Loader Tool Prebind

**Goal:** Close the source-backed maintained-path audit gap by ensuring the generation benchmark,
Sortformer benchmark, embedded-size probe, and paritychecker prebind GGUF KV arena and entry
storage before `model_loader.process_event(...)` starts.

**Requirements:** TENSOR-02, LOAD-02

**Gap Closure:** Closes audit gap `maintained_tool_dispatch_allocation` from
`.planning/milestones/v1.22-MILESTONE-AUDIT.md`.

**Success criteria:**
1. Generation benchmark, Sortformer benchmark, embedded-size probe, and paritychecker allocate or
   reserve required GGUF KV arena and entry storage before dispatching `model_loader.process_event`.
2. The maintained loader callback path no longer calls `resize`, `reserve`, or other heap-allocating
   storage growth while inside the loader actor RTC chain.
3. Focused tests or source-backed checks fail if those maintained lanes regress to dispatch-time
   GGUF KV storage allocation.
4. `TENSOR-02` and `LOAD-02` are restored to satisfied status with source-backed evidence.
5. Scoped quality gates pass; snapshots, benchmark baselines, and model artifacts may be updated
   when required, using the explicit approval granted for this gap-closure work.

### Phase 195: Strict Loader Tensor Closeout

**Goal:** Close the remaining strict audit contradictions by replacing loader tensor result enum
routing with typed internal tensor phase events, removing direct tensor wrapper context reads, and
modeling optional tensor output routing with explicit guards and transitions.

**Requirements:** TENSOR-02, TENSOR-03, TENSOR-04, LOAD-02, LOAD-04

**Gap Closure:** Closes `loader_tensor_outcome_contract_partial`,
`tensor_wrapper_rule_conflict`, and the Phase 194 validation artifact gap from
`.planning/milestones/v1.22-MILESTONE-AUDIT.md`.

**Success criteria:**
1. Loader tensor outcomes are represented by typed `_done` / `_error` internal phase event payloads,
   not `tensor_load_result` enum routing.
2. Tensor plan/apply public wrappers use internal runtime events and never inspect
   `this->context_`.
3. Tensor bind/evict/capture optional `error_out` behavior is selected by explicit guards and
   transition rows, not runtime-indexed helper arrays.
4. Regression tests fail if the audited patterns return.
5. Closeout validation commands pass.

### Phase 196: State Closeout Metadata Repair

**Goal:** Close the final v1.22 audit contradiction by updating state and archive metadata so they
truthfully name Phase 195 as strict runtime closeout and Phase 196 as state metadata repair.

**Requirements:** TENSOR-02, TENSOR-03, TENSOR-04, LOAD-02, LOAD-04

**Gap Closure:** Closes `state_closeout_contradiction` and `state_stale_after_phase_195` from
`.planning/milestones/v1.22-MILESTONE-AUDIT.md`.

**Success criteria:**
1. `.planning/STATE.md` no longer says the milestone stopped after Phase 194.
2. Archived roadmap, requirements, phase artifacts, milestone summary, and audit agree on the
   Phase 195/196 closeout chain.
3. Phase 196 SUMMARY, VERIFICATION, and VALIDATION artifacts map the affected requirements.
4. Source-backed validation commands still pass after metadata repair.

## Closeout Evidence

Phase 194 moved maintained generation benchmark, Sortformer benchmark, embedded-size probe, and
paritychecker GGUF KV storage growth before `model_loader.process_event(...)`. Phase 195 closed the
strict live-source contradictions around loader tensor outcomes and tensor wrapper routing. Phase
196 repaired stale state and archive closeout metadata so all source and planning evidence points
to the same final closure chain.

Source-backed regression coverage now fails if those parse callbacks call `.kv_arena.resize` or
`.kv_entries.resize`, if loader result enum routing returns, if tensor wrappers reintroduce
`bind_or_sink` / `choices[` / `this->context_`, or if domain-boundary guardrails regress.
