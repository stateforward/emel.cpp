# Plan 165-01: Benchmark Actor Boundary Enforcement Closure

<rule_constraints>
- Prefer public state-machine wrappers and event interfaces over actor internals.
- Keep source checks broad enough to cover maintained runner `.cpp` and `.hpp` files.
- Avoid changing benchmark output schemas or fixture semantics.
</rule_constraints>

## Goal

Close `LANE-02` by removing direct benchmark reach-through into actor internals and broadening
source checks so the gap cannot return outside shared runner files.

## Tasks

1. Remove direct action-context and actor-detail reach-through from maintained runner sources.
2. Add a broad source-backed `bench_runner_tests` scan over `tools/bench/**/*.cpp` and `.hpp`
   runner files, excluding the test file's own expected strings.
3. Keep allowed detail surfaces limited to non-actor numeric/kernel/model diagnostic surfaces.
4. Verify generation and diarization JSONL behavior still passes through the public runner binary.
5. Run full unfiltered `bench_runner_tests`, domain-boundary checks, and the changed-file scoped
   quality gate.

- Maintained runner sources now avoid direct actor `action`/`guard` namespaces and prohibited
  actor-owned detail surfaces.
- `bench_runner_tests` now recursively scans maintained `tools/bench` `.cpp` and `.hpp` files for
  actor-internal reach-through patterns.
