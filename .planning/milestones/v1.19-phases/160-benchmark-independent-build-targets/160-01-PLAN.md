# Plan 160-01: Benchmark Independent Build Targets

<rule_constraints>
- Split suite compilation into independent CMake targets without changing runner semantics.
- Preserve `bench_disabled_cases.cpp` compile-definition behavior for filtered suite builds.
- Add focused tests proving suite build wiring is localized.
</rule_constraints>

## Goal

Reorganize `tools/bench` build wiring so benchmark families compile through independent runner
targets while the existing `bench_runner` executable remains the operator entrypoint.

## Tasks

1. Add shared CMake helpers for runner include paths, link inputs, and artifact definitions.
2. Update `add_bench_runner_suite(...)` to create one `bench_runner_suite_<suite>` object target
   per selected suite.
3. Link suite object files into `bench_runner` while preserving disabled-stub compile definitions.
4. Add source checks proving suite targets are independent and registration changes stay local.
5. Validate full-suite and generation-filtered builds.
