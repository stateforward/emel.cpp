# Plan 162-01: Benchmark Manifest Quality-Gate Consumption

<rule_constraints>

## Goal

Wire benchmark dependency manifests into `scripts/quality_gates.sh` so changed files select
relevant benchmark suites conservatively and manifest freshness gaps cannot skip benchmarks.

3. Add manifest write/check freshness checks before benchmark gate decisions.
4. Add `quality_gates_tests.cpp` coverage for manifest consumption and full-gate escalation.
5. Validate focused tests and changed-file scoped quality gate.
