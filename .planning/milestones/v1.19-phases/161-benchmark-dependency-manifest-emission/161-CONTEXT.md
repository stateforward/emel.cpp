# Phase 161: Benchmark Dependency Manifest Emission - Context

**Gathered:** 2026-05-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Add deterministic benchmark dependency manifest emission and freshness checking. This phase should
not wire quality-gate consumption; Phase 162 owns using the manifest to choose benchmark lanes.

</domain>

<decisions>
## Implementation Decisions

### the agent's Discretion
- Mirror the paritychecker dependency manifest pattern for benchmark runners.
- Keep the manifest line-oriented and deterministic for simple shell quality-gate consumption.
- Use `runner=all` records for shared benchmark infrastructure and per-suite records for localized
  runner inputs.

</decisions>
