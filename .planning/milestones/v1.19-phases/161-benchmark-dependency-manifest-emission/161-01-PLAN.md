# Plan 161-01: Benchmark Dependency Manifest Emission

<rule_constraints>
- Add manifest write/check CLI operations without changing normal benchmark execution.
- Keep the checked-in baseline generated from the maintained bench runner binary.
- Cover records, deterministic rendering, freshness states, and operator CLI behavior in tests.
</rule_constraints>

## Goal

Add deterministic per-runner benchmark dependency manifests that name runner source, config,
fixture, model, script, and snapshot inputs for conservative gate impact detection.

## Tasks

1. Add `bench_dependency_manifest.hpp` / `.cpp` with schema, records, render/write/check helpers.
2. Wire `bench_runner` CLI operations for manifest write/check/freshness.
3. Add `tools/bench/dependency_manifest.txt` and schema documentation.
4. Add tests for registered runner coverage, deterministic output, baseline freshness, and CLI
   behavior.
5. Validate focused tests and changed-file scoped quality gate.
