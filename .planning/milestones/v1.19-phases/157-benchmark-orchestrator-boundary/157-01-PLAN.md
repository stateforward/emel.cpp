# Plan 157-01: Benchmark Orchestrator Boundary

<rule_constraints>
- Keep the change scoped to the process shim, shared runner boundary, CMake source list, focused
  source tests, and phase planning artifacts.
- Preserve existing CLI flags, environment variable defaults, output schemas, and exit codes.
- Keep EMEL/reference lane construction unchanged; this phase only moves orchestration ownership.
</rule_constraints>

## Goal

Move benchmark CLI/config/report orchestration behind a runner-owned boundary so `bench_main.cpp`
is no longer the owner of shared benchmark behavior.

## Tasks

1. Add `tools/bench/bench_runner.hpp` with `emel::bench::run_bench_cli(...)`.
2. Move the existing `bench_main.cpp` implementation into `bench_runner.cpp` and expose the CLI
   entrypoint.
3. Replace `bench_main.cpp` with a minimal process shim.
4. Update CMake source wiring for the new runner source/header.
5. Add focused tests proving `bench_main.cpp` delegates and no longer owns the runner body.
