  when no EMEL override is supplied.
- `tools/bench/whisper_emel_parity_runner.cpp` detects legacy `lmgg`, invokes

## Rule Findings

- `src/emel/speech/recognizer/route/token_sequence/actions.hpp` calls
  `emel::model::whisper::detail::build_execution_contract(...)` during recognizer initialization
  action dispatch. That path enters model contract validation and controls whether initialization
  succeeds.
- `src/emel/speech/recognizer/route/token_sequence/guards.hpp` calls
  `validate_tiny_control_tokens(...)` from tokenizer detail code, and the helper result decides
  whether the tokenizer-supported guard passes.

These should be resolved by moving runtime readiness decisions into explicit guards/transitions
and keeping detail helpers out of "what happens next" decisions.
