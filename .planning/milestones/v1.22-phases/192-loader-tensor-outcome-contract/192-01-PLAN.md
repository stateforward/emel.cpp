# Phase 192 Plan 01

## Objective

Close the maintained loader-to-tensor outcome-routing gap by replacing local tensor callback flag
routing with explicit loader bind, plan, and apply decision states.

## Tasks

1. Add a typed same-RTC tensor result payload to `event::load_runtime`.
2. Replace the local `tensor_load_capture` path with bounded tensor result callbacks.
3. Split loader tensor-load orchestration into explicit bind, plan, and apply decision states with
   success, error, and unhandled guards.
4. Add focused regression coverage proving the retired local-capture route is absent and the new
   outcome contract maps callbacks, guards, and errors.
5. Run focused model tests and the changed-file scoped quality gate for loader source, loader
   tests, and updated lint snapshots.

<rule_constraints>
- Do not introduce a new loading strategy or backend-specific loader behavior.
- Do not call the loader actor's own `process_event` from guards, actions, or callbacks.
- Keep tensor actor calls synchronous, bounded, and acyclic.
</rule_constraints>
