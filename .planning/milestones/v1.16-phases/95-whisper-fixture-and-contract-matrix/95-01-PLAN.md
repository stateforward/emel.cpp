# Phase 95 Plan 01: Whisper Fixture And Contract Matrix

**Phase Goal:** Pin the requested model family and validate Whisper tiny GGUF architecture,
tensors, tokenizer, and audio contract.

**Plan Goal:** Lock the narrowed maintained Whisper tiny GGUF variant family ({q8_0, q4_0,
q4_1}), document each fixture's provenance, prove the EMEL loader/contract accepts the
maintained set, prove it rejects non-GGUF reference-lane siblings, and ensure no
whisper.cpp/ggml object can bootstrap EMEL Whisper state.

## Goal-Backward Sketch

To say Phase 95 is done we need:
- README + REQUIREMENTS narrowed and source-backed → fixture manifest tests guard the wording.
- Loader/contract proven on maintained set → focused doctest covers q4_0/q4_1 contract and
  variant-identity ledger.
- Reference-lane isolation → static guardrail that EMEL Whisper source/tests include no
  whisper.cpp/ggml headers.
- Scoped quality gate green → run `scripts/quality_gates.sh` with
  `EMEL_QUALITY_GATES_CHANGED_FILES` scoped to changed files.

## Risk / Notes

- The q4_0/q4_1 fixtures are not staged locally (only `model-tiny-q80.gguf` is present).
  Loader/contract tests for those variants must skip with a `MESSAGE(...)` when missing
  rather than fail; the wording-lock and ledger tests still run.
- The lmgg rejection test must operate entirely on a temporary fixture EMEL controls; it
  must not depend on a real whisper.cpp file existing locally.
- This phase intentionally adds **no** runtime/parity claim. ASR runtime work is Phase 97+.
