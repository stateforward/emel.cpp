# Phase 99 Plan 01: whisper.cpp Parity Lane

<rule_constraints>
## Reference Isolation

- Build `whisper.cpp` only under `build/` and invoke it as an external reference process.
- Do not add reference includes, libraries, state, or objects to `src/`.
- Keep EMEL output generation entirely on EMEL-owned loader/runtime actors.

## Evidence Contract

- Use a stable `whisper_compare/v1` lane record.
- Store EMEL and reference raw JSONL records plus a `whisper_compare_summary/v1` verdict.
- Treat missing tools, missing fixtures, checksum drift, and lane process failures as hard errors.
</rule_constraints>

## Tasks

1. Add pinned reference setup.
   - Fetch `whisper.cpp` tag `v1.7.6`.
   - Verify commit `a8d002cfd879315632a579e73f0148d06959de36`.
   - Download the pinned reference q8_0 model artifact and verify SHA256.
   - Generate a deterministic 16 kHz mono WAV fixture.

2. Add EMEL lane runner.
   - Load the staged EMEL q8_0 fixture through the EMEL GGUF loader.
   - Build the Whisper execution contract.
   - Drive encoder and decoder public actors.
   - Emit a JSONL record with transcript, selected token, digests, checksum, and fixture identity.

3. Add compare wrapper.
   - Build the EMEL runner.
   - Invoke the pinned `whisper-cli` reference lane.
   - Write raw lane records and a summary verdict.
   - Return non-zero only for missing/failed lanes.

4. Wire quality gate.
   - Register `whisper_compare` as a custom quality-gate benchmark suite.
   - Run the scoped gate with `EMEL_QUALITY_GATES_BENCH_SUITE=whisper_compare`.

## Acceptance Criteria

- `scripts/setup_whisper_cpp_reference.sh` pulls and builds the pinned reference backend.
- `scripts/bench_whisper_compare.sh` runs both isolated lanes.
- `build/whisper_compare/summary.json` contains a source-backed verdict for the maintained
  comparison group.
- The scoped quality gate passes through the custom `whisper_compare` suite.
