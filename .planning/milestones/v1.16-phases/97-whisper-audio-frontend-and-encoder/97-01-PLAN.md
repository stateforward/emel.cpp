# Phase 97 Plan 1: Kernel-Owned Whisper Audio Frontend And Full Encoder

<objective>
Implement the maintained Whisper audio frontend and full encoder without shortcuts: mono 16 kHz
PCM request validation, loaded-filter radix-2 STFT/mel preparation, encoder convolution,
positional embedding, attention, and feed-forward execution through EMEL-owned `src/` kernel and
runtime code, ending in deterministic encoder-state evidence for a pinned audio fixture.
</objective>

<rule_constraints>
- **Kernel-owned numeric work (`AGENTS.md`)**: STFT/FFT, mel projection, convolution, layernorm,
  GELU, attention, and dense/quantized projection math must land in `src/emel/kernel/**` or a
  component-local kernel module that explicitly owns Whisper execution before non-kernel wiring.
- **No whole-tensor dequant fallback (`AGENTS.md`)**: q4_0/q4_1/q8_0 encoder weights must use the
  native quantized operand path from Phase 96 or new equivalent kernel paths. Do not materialize
  whole quantized tensors as f32 in the hot path.
- **Runtime choice in SML (`AGENTS.md`)**: validation outcomes, dtype/backend choices, fixture
  support, and request acceptance must be modeled by guards and destination-first transitions in
  `sm.hpp`, not hidden in action/detail branching.
- **No dispatch allocation (`AGENTS.md`, `docs/rules/sml.rules.md`)**: all dispatch-time work must
  use caller/runtime-provided bounded buffers; any heap allocation must happen before
  `process_event(...)`.
- **RTC actor model (`docs/rules/sml.rules.md`)**: no queues, no self-dispatch, no deferred
  mailbox, no re-entrancy, and bounded same-RTC callbacks only.
</rule_constraints>

<verification>
## Must-Haves

1. Public PCM request contract accepts valid mono 16 kHz PCM and rejects invalid audio.
2. Mel prep uses loaded Whisper `mel_filters` and bounded EMEL-owned buffers.
3. Full encoder execution uses maintained `src/` kernel/runtime paths for conv, positions,
   attention, and feed-forward.
4. Deterministic encoder-state digest exists for the pinned model/audio fixture.

## Required Commands

- `cmake --build build/audit-native --target emel_tests_bin`
- `build/audit-native/emel_tests_bin --no-breaks --test-case='*whisper_encoder*,*model_whisper*,*kernel_aarch64_q4*,*kernel_aarch64_q8*'`
- `EMEL_QUALITY_GATES_CHANGED_FILES="<Phase 97 changed files>" scripts/quality_gates.sh`
</verification>

<risk>
- Full encoder completion is a large kernel/runtime phase, not a glue task.
- Existing diarization and embedding audio code proves the project has relevant arithmetic, but
  it is not the owning kernel surface for Whisper. Copying it into a Whisper action would fail the
  user's stated constraint.
- The currently staged q4_0/q4_1 model fixtures are absent, so maintained-family runtime evidence
  may need fixture staging before this phase can claim all three variants.
</risk>
