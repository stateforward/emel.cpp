# Plan 107.1: Speech Tokenizer And Decode Policy Hardening

## Goal

Enforce the maintained tokenizer asset, make Whisper ASR decode policy explicit, and remove
dispatch-time recognizer allocation.

## Tasks

1. Extend tokenizer assets with the pinned tokenizer SHA and reject mismatches during recognizer
   initialize.
2. Add a named Whisper ASR decode policy contract for prompt sequence, English language role,
   transcribe task role, no-timestamps mode, and translate suppression.
3. Move token-sequence route workspaces and child actor allocation into context construction so
   initialize dispatch only binds existing storage.
4. Enforce tokenizer checksum in Whisper compare and benchmark entrypoints.
5. Add focused tests and run the changed-file scoped quality gate with `whisper_compare`.

## Verification

- Focused speech/Whisper tests pass.
- Changed-file scoped quality gate passes with coverage above thresholds.
- Benchmark lane still reports bounded transcript drift, which remains Phase 108 scope.
