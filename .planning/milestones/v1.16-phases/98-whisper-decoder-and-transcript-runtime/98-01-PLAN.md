# Phase 98 Plan 01: Whisper Decoder And Transcript Runtime

<rule_constraints>
## SML And Runtime Rules

- Use destination-first transition rows.
- Model validation and variant outcomes through explicit guards and states.
- Do not store request, output pointers, status, or phase flags in context.
- Keep q8_0/q4_0/q4_1 routing explicit in `sm.hpp`.

## Kernel Ownership

- Extend `src/emel/kernel/whisper/detail.hpp` for decoder numeric work.
- Keep the actor limited to validation, routing, dispatch, and publication.
- Do not add tool-local decoder compute or reference-engine fallback.
</rule_constraints>

## Tasks

1. Add decoder kernel support.
   - Add decoder constants and workspace sizing.
   - Add a one-step decoder path using decoder embedding, positional embedding, self attention,
     encoder cross attention, MLP, final layernorm, full-vocab logits, argmax token selection, and
     deterministic transcript publication.

2. Add `emel::whisper::decoder` actor.
   - Add context, errors, events, guards, actions, detail, and SML model files.
   - Route q8_0, q4_0, and q4_1 through explicit guards/transitions.
   - Validate model contract, encoder-state shape, prompt token, logits capacity, transcript
     capacity, workspace capacity, and unsupported variant paths.

3. Add focused doctests.
   - Drive the staged q8_0 fixture through the existing encoder actor.
   - Feed encoder state into the decoder actor.
   - Assert deterministic token, logits, transcript, digest, and dispatch counters.
   - Assert invalid token and logits capacity errors through public events.

4. Wire build and gates.
   - Add decoder lifecycle tests to `CMakeLists.txt`.
   - Update lint snapshot with approval.
   - Run focused Whisper tests, ctest shard plus lint snapshot, and scoped quality gate.

## Acceptance Criteria

- `emel::whisper::decoder::sm` exists and exposes a public `event::decode`.
- Decoder tokens and logits are produced through EMEL-owned `src/` code.
- Transcript output is deterministic for the maintained fixture.
- Malformed token/capacity requests report explicit errors.
- Runtime behavior choice follows SML guards/transitions.
