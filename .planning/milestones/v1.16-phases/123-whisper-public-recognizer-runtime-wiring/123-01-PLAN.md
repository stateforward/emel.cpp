# Plan 123.1: Whisper Public Recognizer Runtime Wiring

<objective>
Make the public speech recognizer actor initialize and recognize the maintained Phase 99 Whisper
model/tokenizer/audio path while preserving the generic recognizer domain boundary.
</objective>

<rule_constraints>
- Keep `src/emel/speech/recognizer/**` and `tests/speech/recognizer/**` free of Whisper names so
  `scripts/check_domain_boundaries.sh` continues to enforce generic recognizer cleanliness.
- Model backend availability, tokenizer support, model support, and recognition readiness as
  explicit recognizer guards/transitions.
- Do not allocate during recognizer dispatch; recognition scratch storage must be caller-owned
  event payload.
- Do not copy per-dispatch request, output, phase, status, or error data into recognizer context.
- Do not expose `emel::model::whisper` contracts from speech encoder/decoder public runtime
  events, contexts, guards, or actions.
- Drive child encoder/decoder actors only through their public events and `process_event(...)`.
</rule_constraints>
