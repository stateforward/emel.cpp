# Plan 114-01: Whisper Runtime Surface Contract Repair

<objective>
Reconcile the maintained v1.16 Whisper ASR runtime contract with live source. The selected contract
is the speech-owned encoder/decoder actor pair plus Whisper tokenizer policy, not the generic
recognizer actor.
</objective>

<rule_constraints>
- Do not add Whisper implementation under the generic recognizer.
- Do not expose `emel::model::whisper` contracts from speech encoder/decoder public runtime
  events, contexts, guards, or actions.
- Keep compare and benchmark lanes split between EMEL-owned runtime and `whisper.cpp` reference.
- Preserve exact transcript parity and the benchmark performance gate.
</rule_constraints>
