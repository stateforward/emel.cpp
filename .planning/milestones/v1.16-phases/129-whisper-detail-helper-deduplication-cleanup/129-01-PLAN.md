# Plan 129.1: Whisper Detail Helper Deduplication Cleanup

<rule_constraints>

- Keep encoder detail encoder-owned.
- Keep decoder runtime execution decoder-owned.
- Avoid new shared detail unless a helper is genuinely reused by multiple production owners.
- Preserve SML behavior-selection and domain-boundary checks.

</rule_constraints>

## Tasks

1. Remove duplicate decoder helpers from encoder detail.
   - Delete decoder token constants, `decode_policy_runtime`,
     `required_decoder_workspace_floats`, decoder cross-cache/logit helpers,
     timestamp-aware token selection, and `run_decoder_sequence` from encoder detail.
   - Leave encoder audio/encoder helpers untouched.

2. Move helper tests to the decoder owner.
   - Remove timestamp helper coverage from `tests/speech/encoder/whisper/detail_tests.cpp`.
   - Add decoder-owned timestamp helper coverage to
     `tests/speech/decoder/whisper/lifecycle_tests.cpp`.
   - Add a source regression that encoder detail does not contain decoder helper names.

3. Verify ownership and maintained behavior.
   - Rebuild `emel_tests_bin` after adding the decoder detail test source.
   - Run focused encoder/decoder Whisper tests.
   - Run source greps for decoder production ownership and encoder-detail helper removal.
   - Run SML/domain checks and maintained compare/benchmark checks.
