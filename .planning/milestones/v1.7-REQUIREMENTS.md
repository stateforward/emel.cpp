# Requirements: EMEL

**Defined:** 2026-03-29
**Core Value:** Prove real end-to-end behavior with explicit SML orchestration and parity-oriented
verification before widening API surface or model scope.

## v1 Requirements

### Prefill Orchestration

- [x] **PREF-01**: Generator owns an explicit `src/emel/generator/prefill` orchestration machine
  for prefill slots, snapshot, compute dispatch, and prefill handoff.
- [x] **PREF-02**: Prefill request-scoped orchestration data stays on typed runtime/internal events
  instead of being mirrored into generator context flags or phase fields.

### Routing Transparency

- [x] **ROUT-01**: Prefill compute routing is collapsed into explicit prefill contracts and no
  longer exists as a repeated flash/nonflash x materialized/preselected x scalar/chunk4 cartesian
  matrix in one top-level generator table.
- [x] **ROUT-02**: Prefill route selection remains explicitly modeled through guards, states, and
  transitions, with no hidden runtime branching inside generator actions or detail helpers.

### Maintainability And Proof

- [x] **ARCH-01**: The top-level generator surface is materially smaller and easier to inspect after
  the prefill extraction, with prefill-specific boilerplate reduced.
- [x] **VERIFY-01**: Maintained generator, paritychecker, benchmark, and quality-gate coverage
  stayed green on the current Llama and canonical Qwen slices, with the user explicitly waiving
  unrelated broad benchmark snapshot regressions outside `generator/prefill` for v1.7 closeout.

## v2 Requirements

### Further Generator Decomposition

- **DECODE-01**: Extract generator decode orchestration into a smaller generator-owned machine
  after the prefill extraction pattern is proven.
- **ATTN-01**: Revisit attention-family decomposition such as `flash` vs `nonflash` through
  `sm_any` only after the prefill routing matrix is no longer the main duplication source.

### Product Scope

- **MODEL-02**: Broaden beyond the canonical Qwen3-0.6B fixture to additional Qwen architectures
  or quantizations once the first slice is proven and benchmarked.
- **COND-02**: Add richer Qwen chat or tool-calling request surfaces only after the canonical
  maintained slice has an explicit and stable conditioning contract.

### Performance And Policy

- **GEN-03**: Optimize remaining generator-side hot spots after the maintained decomposition work
  is correct and parity-backed.
- **BENCH-07**: Revisit whether noisy benchmark drift should become a blocking repo gate once the
  maintained compare surfaces are stable enough to justify it.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Decode submachine extraction in v1.7 | Prove the prefill extraction pattern first before touching sampling/render/loop orchestration. |
| Separate session/runtime actor redesign | This milestone decomposes `generator`, not ownership around it. |
| Attention-family `sm_any` extraction in v1.7 | Attention mode is only one axis of the current duplication and should not hide unresolved top-level routing. |
| Broader Qwen-family or multi-fixture support | Keep the acceptance boundary on the current maintained Llama and canonical Qwen anchors. |
| Benchmark gate hardening | Warning-only benchmark debt is separate from this generator refactor. |
| Hidden control-flow shortcuts in actions/detail helpers | The milestone must preserve explicit behavior modeling rather than trading structure for convenience. |
