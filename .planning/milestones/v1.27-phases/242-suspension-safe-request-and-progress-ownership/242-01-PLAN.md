# Phase 242 Plan 01: Suspension-Safe Request and Progress Ownership

**Status:** Ready
**Requirements:** AIO-03, OWN-01, OWN-02, OWN-03, OWN-04
**Date:** 2026-05-09

## Goal

Add the stable request/progress ownership contract and validation graph needed before async I/O can
suspend or resume.

## Tasks

1. Add async request/progress storage types with explicit lifetime comments.
2. Replace the phase-241 immediate unsupported route with guard-modeled validation states.
3. Keep actor context empty and keep callbacks same-RTC only.
4. Add tests for valid-but-unsupported, invalid source, invalid target, invalid progress, callback
   handling, context cleanliness, and scheduler contract proof.
5. Build, run focused tests, and run changed-file scoped quality gates.

## Acceptance Checks

- Valid async requests reach the fail-closed `unsupported_strategy` path only after source, target,
  progress, callback, and scheduler contracts are satisfied.
- Invalid request/progress cases publish deterministic validation errors.
- No stack-backed request, span, target, callback, or progress pointer is stored in context.
- The selected async scheduler still proves FIFO, single-consumer, run-to-completion semantics.
