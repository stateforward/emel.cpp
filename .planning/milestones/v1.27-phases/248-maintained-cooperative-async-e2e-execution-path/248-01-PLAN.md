# Phase 248 Plan

## Goal

Close `PERF-01` by making the maintained model-loader/generation path execute
`cooperative_async` through public loader/tensor contracts and publish measured evidence.

## Tasks

1. Add a public `io/loader` cooperative async route backed by an injected
   `emel::io::async::sm`, including single-tensor and batch behavior.
2. Preserve actor-boundary rules: tools and model-loader code must continue using public
   `io/loader` / `model/tensor` contracts and must not include async actor internals.
3. Wire maintained generation benchmark fixture setup with an async actor so
   `EMEL_MODEL_LOAD_IO_STRATEGY=cooperative_async` no longer fails with
   `io_strategy_unavailable`.
4. Add focused tests proving direct `io/loader` async dispatch, model-loader async success, and
   maintained tool public-contract boundaries.
5. Update `247-PERFORMANCE.md`, `REQUIREMENTS.md`, `ROADMAP.md`, `STATE.md`, and the milestone
   audit only after source-backed async execution evidence exists.
6. Run focused tests and changed-file scoped quality gates without benchmark-regression override.

## Constrained-RAM Proof

The final report must explain that the benefit for large models in constrained RAM is bounded
window progress: the maintained path copies tensor windows through the async actor into
tensor-owned residency rather than treating unsupported async as evidence or requiring the full
model payload to be staged as the async result.
