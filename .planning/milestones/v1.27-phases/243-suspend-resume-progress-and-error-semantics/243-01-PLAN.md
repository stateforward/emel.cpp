# Phase 243 Plan 01: Suspend/Resume Progress and Error Semantics

**Status:** Ready
**Requirements:** AIO-04, AIO-05, AIO-06, TST-02
**Date:** 2026-05-09

## Goal

Make cooperative async loading advance bounded progress through public dispatch and publish explicit
partial, success, and error outcomes.

## Tasks

1. Extend async storage with caller-owned source span and per-dispatch chunk size.
2. Add progress outcome events and callbacks for partial progress.
3. Add SML states/guards for cancellation, partial progress, terminal progress, and terminal errors.
4. Implement chunk-copy actions with no hidden behavior selection.
5. Add focused tests for partial/resume/success/cancel/error paths and state inspection.
6. Run focused tests, lint, and changed-file scoped quality gates.

## Acceptance Checks

- A valid dispatch copies at most one configured chunk and publishes partial progress when bytes
  remain.
- A later dispatch resumes from caller-owned progress and publishes terminal success only after the
  logical byte span is complete.
- Cancellation and validation/source/progress failures publish deterministic terminal errors.
- Tests prove behavior through public dispatch and SML state inspection.
