# Phase 244 Plan 01: Tensor-Owned Async Integration Graph

**Status:** Ready
**Requirements:** TNX-01, TNX-02, TNX-03, TST-03
**Date:** 2026-05-09

## Goal

Route cooperative async loading through the tensor actor without leaking tensor residency ownership
into the I/O strategy.

## Tasks

1. Add tensor async request/outcome events and status plumbing.
2. Add async strategy injection to tensor context and wrapper constructors.
3. Add guards/actions/transition rows for async dispatch, partial progress, terminal success, and
   errors.
4. Add tests for tensor async partial progress, terminal residency commit, and failure paths.
5. Run focused tests, lint, and changed-file scoped quality gates.

## Acceptance Checks

- Tensor initiates async loading only by dispatching public `emel::io::async` events.
- Partial async progress is visible to tensor callers and leaves tensor lifecycle unbound.
- Terminal async success commits tensor lifecycle resident and records buffer/window bytes.
- Async failures publish tensor-owned `_error` outcomes without mutating residency.
