# Phase 240 Plan 01: co_sm Wrapper, Scheduler, and Allocation Proof

**Status:** Ready
**Requirements:** CO-02, CO-03, CO-04, CO-05, TST-01
**Date:** 2026-05-09

## Goal

Introduce the project-owned opt-in `emel::co_sm` wrapper and prove baseline scheduler/allocation
behavior before the async I/O strategy lands.

## Tasks

1. Include upstream `stateforward/sml/utility/co_sm.hpp` in `src/emel/sm.hpp`.
2. Add EMEL policy aliases for inline/FIFO schedulers and coroutine allocator policies.
3. Add `fixed_coroutine_allocator` that never falls back to heap allocation.
4. Add `emel::co_sm` specializations for contextless and contextful machines.
5. Add focused `co_sm` doctests to `tests/sm/sm_policy_tests.cpp`.
6. Run focused tests and changed-file quality gates.

## Acceptance Checks

- `emel::co_sm` compiles as an opt-in wrapper parallel to `emel::sm`.
- Scheduler policies prove FIFO, single-consumer, run-to-completion contracts.
- Fixed allocator returns `nullptr` instead of heap fallback when exhausted or oversized.
- Immediate async dispatch completes deterministically and uses zero coroutine allocator calls.
- Context injection and state inspection work through the public wrapper.
