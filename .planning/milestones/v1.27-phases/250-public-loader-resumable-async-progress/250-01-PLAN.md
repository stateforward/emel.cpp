# Phase 250 Plan

<rule_constraints>
Runtime behavior choice stays in guards and transition tables. Loader/model-loader actions may
dispatch exactly the already-selected public event path, but must not hide cooperative async
completion behind local drain loops.
</rule_constraints>

## Goal

Preserve bounded cooperative async partial progress across the maintained `io/loader` and
model-loader boundary, closing `AIO-04`, `TNX-03`, `PERF-01`, and `INT-01`.

## Tasks

1. Extend public loader cooperative async contracts with caller-owned progress storage and
   progress result events for single and maintained batch loading.
2. Remove local async drain loops from loader actions; each public dispatch advances only a
   bounded cooperative chunk for each selected async request.
3. Route partial, terminal success, unsupported, and failure outcomes through explicit loader
   and model-loader guards/states.
4. Keep `model/tensor` residency ownership unchanged and prove tensor-visible async progress,
   success, and failure stay public.
5. Update generation benchmark/probe reporting so cooperative async evidence names partial
   progress and terminal completion honestly.
6. Add focused doctests for loader partial progress and maintained model-loader async progress.
7. Run focused tests, generation benchmark/probe if available, and changed-file scoped quality
   gates with the generation benchmark suite.
