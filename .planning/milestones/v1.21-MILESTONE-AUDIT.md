## Tech Debt

- Cold workspace selective optimization depends on pre-existing
  `build/paritychecker_zig/paritychecker` and `build/bench_tools_ninja/bench_runner` binaries for
  manifest freshness checks. If they are missing, the implementation correctly fails closed to full
  relevant parity/benchmark lanes, but the first run will not realize the selected-runner speedup.
- The focused quality-gate tests are mostly static source-contract assertions. That catches
  accidental removal of the intended shell contracts, but it is weaker than behavioral tests that
  execute impact resolution, fallback, selected benchmark execution, and parallel reporting.
- Phase 184 records selected parity E2E evidence but not a localized benchmark-only quality-gate
  run that proves `scripts/quality_gates.sh` reaches `scripts/bench.sh --suite=<runner>` end to end.
- `node .codex/get-shit-done/bin/gsd-tools.cjs validate health` remains degraded by pre-existing
  stale worktree warnings and a missing `workflow.ai_integration_phase` config default.
