# Milestone v1.2: Flash Attention

**Status:** ✅ SHIPPED 2026-03-22
**Phases:** 10-13
**Total Plans:** 13

## Overview

v1.2 shipped a narrow but complete flash-attention milestone on top of the canonical Llama-68M
generation slice: the repo now has a real EMEL-owned flash-attention data path in the shipped
generator/runtime chain, paritychecker proves that path against a fetched upstream reference, graph
and generator runtime lifecycle authority is hard-cut over to `emel::tensor::sm`, and the existing
benchmark workflow publishes maintained flash evidence with a preserved pre-flash baseline plus
generated docs.

## Milestone Summary

**Decimal Phases:**

- Phase 12.1: Enforce SML Tensor Lifecycle Orchestration (inserted after Phase 12 once flash
  runtime work exposed a lifecycle bypass in the shipped graph/generator path)

**Key Decisions:**

- Keep the milestone inside the existing paritychecker plus bench surfaces instead of widening into
  new runtime or API work.
- Insert Phase 12.1 rather than shipping flash benchmark claims over a lifecycle-bypass runtime
  path.
- Preserve benchmark evidence on the existing compare/snapshot/docs workflow and require explicit
  approval before mutating checked-in benchmark artifacts.
- Publish benchmark reference identity and flash proof as durable comments ahead of unchanged
  compare rows.

**Issues Resolved:**

- The canonical generator path now has a real EMEL-owned flash-attention execution seam.
- Paritychecker and bench now both publish truthful fetched-reference identity instead of relying
  on repo-local `tmp/llama.cpp`.
- Graph and generator runtime lifecycle truth is now owned by `emel::tensor::sm`.
- The maintained benchmark workflow now preserves pre-flash evidence separately from current flash
  publication.

**Issues Deferred:**

- Benchmark-policy hardening for noisy low-iteration snapshot smoke remains outside this milestone.
- Flash benchmark claims remain intentionally narrow to the canonical CPU-hosted Llama-68M slice
  and the maintained short compare case.
- Broader flash operand/model coverage and backend-specific optimized kernels are deferred to a
  future milestone.

**Technical Debt Incurred:**

- The missing `12.1-VERIFICATION.md` report had to be backfilled during milestone closeout even
  though runtime evidence already existed.
- Bench configure/build still surfaces non-blocking OpenMP, FetchContent, and environment warnings.
- `scripts/quality_gates.sh` still treats benchmark drift as warning-only repo policy.
