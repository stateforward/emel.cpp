# Phase 12.1 Plan 02: Processor Lifecycle Gate Summary

**Graph processor execution now gates kernel dispatch on tensor lifecycle truth, publishes outputs through the tensor actor, and blocks tensor reuse until release is formally recorded, with generator manifests cut over to the same contract.**

## Accomplishments
- Added explicit pre-kernel lifecycle gating in `graph::processor` so required inputs must already be filled and publish targets must already be empty before kernel dispatch can occur.

- Fixed the generator lifecycle manifests and sequence-reset behavior so native flash generation still satisfies the enforced tensor lifecycle contract.

## Decisions Made
- The processor lifecycle gate enforces two truths before kernel execution: required inputs must be filled and publish targets must be reusable.
- Publish and release remain explicit processor phases instead of being hidden inside kernel or extract callbacks.
- Generator decode treats KV cache tensors as ongoing filled state, not empty-to-filled publish targets on every step.
