# Requirements: EMEL

**Defined:** 2026-04-17
**Core Value:** Prove real end-to-end behavior with explicit SML orchestration and parity-oriented
verification before widening API surface or model scope.

## v1 Requirements

### Reference Backend Architecture

- [x] **REF-01**: Operator can select a reference backend for parity or benchmark runs without
  changing the EMEL lane implementation.
- [x] **REF-02**: Python and C++ reference backends emit one canonical comparison contract for
  embeddings, timings, and backend metadata.
- [x] **ISO-01**: The EMEL lane remains isolated from reference-engine model, tokenizer, cache,
  and runtime objects.

### Python Reference Backends

- [x] **PY-01**: Operator can run at least one Python reference backend for embedding comparison
  through the shared comparison contract.
- [x] **PY-02**: Python environment or backend failures surface explicit, reproducible errors
  without corrupting the EMEL result lane.

### C++ Reference Backends

- [x] **CPP-01**: Existing C++ reference lanes can run through the same pluggable backend contract
  and produce the canonical comparison output.
- [x] **CPP-02**: C++ backend-specific setup remains confined to the reference lane and does not
  leak into `src/` runtime code or the EMEL compute path.

### Unified Compare Workflow

- [x] **CMP-01**: Operator can run one consistent EMEL-vs-reference compare workflow regardless of
  backend language.
- [x] **CMP-02**: Compare artifacts publish backend identity, fixture identity, and enough
  similarity and configuration metadata to reproduce results across inference engines.

## v2 Requirements

### Follow-On Scope

- **SDK-01**: EMEL exposes a stable public plugin SDK for third-party reference backends outside
  the repo.
- **REM-01**: Compare workflows support remote or service-hosted reference engines.
- **GEN-01**: The pluggable reference architecture extends beyond embeddings to generation and
  non-embedding inference surfaces.

## Out of Scope

| Feature | Reason |
|---------|--------|
| New public embedding C ABI or broad CLI API commitments | This milestone is about internal compare architecture, not public API surface |
| Remote HTTP or service-hosted reference engines | Keep the first pluggable design local and deterministic |
| Non-embedding generation parity scope | Keep the milestone bounded to the maintained embedding compare lane |
| Broad new `src/` runtime support added only to satisfy a reference backend | Reference adapters must not drive EMEL runtime scope creep |
| Shared model, tokenizer, cache, or runtime objects between EMEL and reference lanes | The compare architecture must preserve explicit lane isolation |
