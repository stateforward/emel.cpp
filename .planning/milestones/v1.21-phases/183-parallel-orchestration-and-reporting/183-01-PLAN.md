# Phase 183 Plan 01

<rule_constraints>
- Parallel lanes must have independent artifacts or run after the shared build step.
- Lane failures must be visible in aggregate output.
- Serial execution must remain available for debugging.
</rule_constraints>

## Objective

Add deterministic parallel orchestration for independent heavy quality-gate lanes.

## Tasks

1. Add `EMEL_QUALITY_GATES_PARALLEL` policy parsing.
2. Suppress child timing writes with `EMEL_QUALITY_GATES_PARALLEL_CHILD=1`.
3. Add `start_parallel_step`, `finish_parallel_steps`, and `run_parallel_quality_group`.
4. Run benchmark, coverage, parity, and fuzz lanes in parallel after the serial build preflight.
5. Replay lane logs in a stable order with status and duration.
6. Add static tests covering parallel orchestration and failure capture.

## Verification

- `bash -n scripts/quality_gates.sh`
- `build/bench_tools_ninja/quality_gates_tests`
- changed-file scoped quality gate output contains parallel log begin/end records and timing data.

## Done Condition

Independent heavy lanes run concurrently with deterministic logs, aggregate timing, and correct
failure handling.
