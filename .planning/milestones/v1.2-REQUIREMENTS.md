# Milestone v1.2 Requirements: Flash Attention

**Status:** ✅ SHIPPED 2026-03-22
**Audit:** passed
**Source milestone:** v1.2

## Final Outcome

v1.2 shipped all 10 planned requirements for the canonical Llama-68M flash-attention slice. The
milestone added an EMEL-owned flash-attention kernel and generator path, proved that path through
the normal paritychecker surface, hard-cut tensor lifecycle orchestration over to
`emel::tensor::sm`, and published truthful benchmark evidence with a preserved pre-flash baseline
artifact plus generated docs.

## v1 Requirements

### Flash Kernel

- [x] **FLASH-01**: The canonical Llama-68M attention step can execute through an EMEL-owned
  `op_flash_attn_ext` implementation under `src/emel/kernel`.
- [x] **FLASH-02**: The flash-attention operator uses persistent workspace or buffers and does not
  introduce allocation in the inference hot path.

### Generation Runtime

- [x] **GEN-01**: The shipped canonical generation path under `src/emel/generator` dispatches
  through flash attention for supported Llama-68M requests.
- [x] **GEN-02**: Unsupported requests publish explicit deterministic behavior when flash attention
  is not selected or cannot run; the runtime must not silently claim flash-path execution.

### Parity And Verification

- [x] **PAR-01**: `tools/paritychecker --generation` can prove that the flash-attention path
  executed on `tests/models/Llama-68M-Chat-v1-Q2_K.gguf`.
- [x] **PAR-02**: The canonical flash-attention generation result remains parity-stable against an
  aligned `llama.cpp` reference configuration.
- [x] **VER-01**: Tests cover shared-kernel correctness, generator adoption, and negative
  selection or fallback behavior for the flash-attention path.

### Benchmarking

- [x] **BENCH-01**: `tools/bench` can run the canonical flash-attention path through the existing
  compare workflow on the same maintained workload contract.
- [x] **BENCH-02**: Benchmark output and maintained artifacts clearly distinguish flash-attention
  evidence from the previous non-flash baseline.
- [x] **BENCH-03**: Canonical flash-attention benchmark results show measurable improvement over
  the current EMEL non-flash canonical baseline on at least one maintained case.

## Outcome Notes

- Phase 12.1 was inserted after Phase 12 because flash runtime work had left a tensor lifecycle
  bypass in graph/generator orchestration. The inserted phase is what made the shipped runtime
  architecture contract truthful again before benchmarking.
- Benchmark publication stayed on the existing `scripts/bench.sh --compare`,
  `--compare-update`, and `scripts/generate_docs.sh` surfaces, and checked-in benchmark artifact
  changes remained blocked on explicit user approval.
- The milestone audit passed with 10/10 requirements satisfied. Remaining issues are narrow
  non-blocking tech debt around benchmark-policy noise, tooling warnings, and planning-process
  hygiene rather than requirement failures.
