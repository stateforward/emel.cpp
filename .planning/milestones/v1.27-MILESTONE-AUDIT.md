## Source-Backed Maintained Path

The maintained generation fixture constructs public model-loader load events, binds
`cooperative_async`, supplies public loader progress storage, and repeatedly dispatches
`model_loader.process_event(load_ev)` until done. `model/loader` has explicit cooperative async
load and resume states, routes into public `io/loader` batch dispatch, and returns to `ready` on
partial progress. `io/loader` selects `cooperative_async` via guards, dispatches public
`io/async` progress ticks, and publishes partial/done/error outcomes explicitly.

The generation benchmark records load-profile metadata only when the actually used strategy is
`cooperative_async`. It labels the profile as `constrained_ram_emulation` and
`source_residency=setup_time_fixture_file_image_and_full_target_storage`, so it does not claim true
larger-than-RAM source residency. The benchmark runner prints that metadata in
`# generation_load_profile:` lines.

Guardrail tests confirm tools do not include `emel/io/async/` internals or reach directly into
async actor `load_window` implementation details.
