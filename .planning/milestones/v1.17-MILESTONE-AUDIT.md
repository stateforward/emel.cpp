# Milestone v1.17 Audit

## Verdict

**Status:** passed

Milestone v1.17 is ready for completion. The canonical generative text actor now lives under
`text/generator`, the public parity and benchmark entrypoints drive the maintained actor through
`process_event(...)`, and the reopened source-backed closeout chain has closed the final graph
callback outcome ownership gap.

The previous failed audit blocker was not waived or bypassed. Phase 147 changed the maintained
generator path so graph validation, bind, and extract callbacks used by the already accepted
generator route are guard-accepted execution callbacks. They no longer use `request_plan`,
`check_backend`, output pointer null branching, `expected_outputs`, token/position counts,
`*err_out`, or invalid error codes to choose graph processor success/error behavior. Those
readiness decisions are now represented by explicit generator guards and SML transitions before
the graph compute dispatch.

## Required Remedy Closed

The prior failed audit required the generator to stop relying on detail callback outputs to decide
graph validation, bind, and extract outcomes. That remedy is closed:

- `actions.hpp` now wires guarded callback variants only.
- `guards.hpp` owns materialized output readiness, bound request capacity, prefill/decode
  readiness, invalid routes, and backend-unavailable routes.
- `detail.hpp` guarded callbacks ignore `err_out`, avoid validation-path branching, and return
  true on the maintained already-accepted route.
- Regression tests scan maintained action wiring and callback spans so the old hidden outcome path
  cannot be reintroduced silently.
- Maintained parity, benchmark, and embedded-size tools drive only the public actor surface.

No blocking milestone gaps remain.
