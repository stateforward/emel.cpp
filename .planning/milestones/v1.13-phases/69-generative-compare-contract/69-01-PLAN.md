# Phase 69 Plan 01

<rule_constraints>
- Add the first generation compare contract in `tools/bench`; do not change `src/emel/generator`
  orchestration or broaden maintained runtime scope.
- Preserve separate EMEL and reference lane identity even when both lanes emit the same schema.
- Keep the existing text benchmark/compare output working so this phase is additive, not a
  reporting rewrite.
</rule_constraints>

## Objective

Define one canonical pluggable compare contract for generative workloads and make the maintained
EMEL and built-in C++ reference generation lanes emit it in JSONL mode without violating lane
isolation.

## Tasks

1. Add a shared `generation_compare/v1` record contract with schema/versioning, lane identity,
   workload identity, timing fields, output checksums, and optional dumped output files.
2. Update the maintained generation bench surface so `--mode=emel` and `--mode=reference` can emit
   machine-readable generation compare records while preserving the existing human-readable output.
3. Add focused regression coverage proving that JSONL mode emits the canonical generation schema
   and keeps the current text mode intact.

## Verification

- `cmake --build build/bench_tools_ninja --parallel --target bench_runner bench_runner_tests`
- `ctest --test-dir build/bench_tools_ninja --output-on-failure -R bench_runner_tests`

## Done Condition

The phase is complete when the maintained EMEL and C++ reference generation lanes can emit the
same `generation_compare/v1` machine-readable records without changing `src/` runtime behavior or
removing the existing human-readable benchmark surface.
