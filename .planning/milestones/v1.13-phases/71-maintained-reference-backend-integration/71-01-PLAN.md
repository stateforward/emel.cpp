# Phase 71 Plan 01

<rule_constraints>
- Keep all new backend selection, build, and environment handling in `tools/bench` and `scripts/`.
- Reuse the maintained generation bench JSONL surface rather than adding new `src/` runtime
  behavior.
- Failures must become explicit `generation_compare/v1` error rows and compare-summary entries.
</rule_constraints>

## Objective

Put the first maintained non-EMEL generative backend behind a manifest-driven generation compare
surface so EMEL and the selected reference backend can be run independently and summarized
reproducibly.

## Tasks

1. Add a generation reference-backend manifest contract and the first maintained C++ llama.cpp
   wrapper script.
2. Add a repo-owned generation compare driver that runs the EMEL lane and the selected reference
   backend independently, writes raw JSONL artifacts, and synthesizes explicit compare-summary
   error states.
3. Add focused regression coverage proving the manifest-driven backend integration and error-record
   behavior without depending on live long-running generation runs.

## Verification

- `cmake --build build/bench_tools_ninja --parallel --target generation_compare_tests`
- `ctest --test-dir build/bench_tools_ninja --output-on-failure -R generation_compare_tests`

## Done Condition

The phase is complete when a maintained generation reference backend can be selected through a
manifest and run through the shared compare contract, and backend setup/errors remain confined to
tooling surfaces instead of leaking into the EMEL runtime lane.
