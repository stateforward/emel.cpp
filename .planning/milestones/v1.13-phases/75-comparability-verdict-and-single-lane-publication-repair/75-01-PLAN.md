# Phase 75 Plan: Comparability Verdict And Single-Lane Publication Repair

<rule_constraints>
## Applicable Rules

- EMEL/reference lanes must remain isolated; reference backend selection cannot alter the EMEL
  generation lane implementation.
- Compare publication must truthfully distinguish non-comparable workloads from lane failures.
- Workload and verdict repair belongs in benchmark tooling, manifests, docs, and tests for this
  phase.
</rule_constraints>

## Goal

Make the operator-facing generation compare workflow publish real maintained single-lane
non-comparable workloads and reject material comparability metadata drift before output comparison.

## Tasks

1. Add maintained single-lane workload proof.
   - Create an LFM2 single-lane workload manifest using the checked-in local LFM2 fixture.
   - Register the manifest in the maintained generation workload list.
   - Keep the manifest explicitly marked `comparison_mode="single_lane"` and `comparable=false`.

2. Repair wrapper verdict behavior.
   - Teach `generation_compare.py` to inspect the selected workload manifest.
   - Skip reference backend build/run only when the selected workload is non-parity or
     non-comparable.
   - Preserve an empty `raw/reference.jsonl` and publish `single_lane_emel_workload` rather than
     a reference lane error.

3. Expand comparability metadata checks.
   - Copy model/fixture/prompt/formatter/sampling/stop/seed/token-budget metadata into summary
     groups.
   - Mark metadata mismatches as `non_comparable` before exact-match or bounded-drift checks.
   - Preserve existing mismatch reason names for previously covered fields.

4. Add and run regression proof.
   - Add synthetic mismatch tests for ignored metadata fields.
   - Add a wrapper-level single-lane workflow test using the maintained LFM2 fixture.
   - Run focused tests, CTest, and the full quality gate.

## Expected Evidence

- `generation_compare_tests` passes directly and through CTest.
- The single-lane wrapper test proves real operator-path publication, not synthetic JSONL-only
  scaffolding.
- `./scripts/quality_gates.sh` exits `0`.
- Phase verification maps `WRK-03`, `CMP-03`, and `PRF-01` to concrete code and command evidence.
