## Current Milestone

### v1.7 Generator Prefill Submachine Decomposition

**Milestone Goal:** Decompose the generator’s prefill orchestration into a smaller explicit machine
inside the `generator` domain while keeping request-scoped behavior modeled via typed events,
guards, and transitions instead of hidden action/detail control flow.

**Scope Guardrails:**
- Keep the milestone narrow to prefill decomposition plus the top-level prefill compute-contract
  collapse.
- Keep `generator` as the owning domain; do not redesign this as a separate session/runtime actor
  milestone.
- Preserve maintained Llama/Qwen parity, benchmark, snapshot, and docs behavior while refactoring.
- Do not hide route choice in actions/detail helpers or broaden the milestone to decode extraction,
  richer Qwen surfaces, or benchmark gate policy.

## Phase Details

### Phase 30: Generator Prefill Contract Collapse
**Goal**: Replace the repeated top-level prefill routing matrix with explicit request-scoped
prefill contracts and shared prefill compute/result states.
**Depends on**: Phase 29
**Requirements**: ROUT-01, ROUT-02
**Success Criteria** (what must be TRUE):
  1. Prefill route choice resolves into explicit request-scoped prefill contracts carried on typed
     runtime/internal events instead of being spelled out as repeated state families in the
     top-level generator table.
  2. The top-level generator no longer repeats the old flash/nonflash x
     materialized/preselected x scalar/chunk4 prefill matrix as separate control-flow families.
  3. Prefill route selection stays explicitly modeled through guards/states/transitions, with no
     hidden routing logic added to actions or detail helpers.
**Plans**: 2 plans

Plans:
- [x] 30-01: Define request-scoped prefill contracts and internal handoff events.
- [x] 30-02: Collapse top-level prefill routing around shared compute/result states.

### Phase 31: Generator Prefill Submachine Extraction
**Goal**: Move prefill orchestration into an explicit generator-owned `prefill` machine.
**Depends on**: Phase 30
**Requirements**: PREF-01, PREF-02
**Success Criteria** (what must be TRUE):
  1. `src/emel/generator/prefill` owns prefill slots, snapshot capture, compute dispatch, and
     prefill handoff as an explicit machine inside the generator domain.
  2. The parent generator delegates prefill through an explicit machine boundary and typed events
     rather than direct helper calls or copied phase flags.
  3. Prefill request-scoped data remains on typed runtime/internal events instead of being added to
     generator context as orchestration state.
**Plans**: 2 plans

Plans:
- [x] 31-01: Create the generator/prefill machine boundary, events, and context seam.
- [x] 31-02: Integrate the parent generator with generator/prefill without behavior drift.

### Phase 32: Prefill Surface Shrink And Proof
**Goal**: Reduce top-level generator boilerplate and close the milestone with maintained proof.
**Depends on**: Phase 31
**Requirements**: ARCH-01, VERIFY-01
**Success Criteria** (what must be TRUE):
  1. The top-level generator surface is materially smaller and easier to inspect after the prefill
     extraction, with prefill-specific boilerplate/accessor burden reduced.
  2. Maintained generator, paritychecker, benchmark, and quality-gate coverage remain green on the
     current Llama and canonical Qwen slices.
  3. The milestone lands without broadening into decode extraction, attention-family `sm_any`, or
     separate session/runtime redesign.
**Plans**: 2 plans

Plans:
- [x] 32-01: Shrink the top-level generator surface after prefill extraction.
- [x] 32-02: Refresh maintained regression and gate proof for the new prefill machine boundary.
