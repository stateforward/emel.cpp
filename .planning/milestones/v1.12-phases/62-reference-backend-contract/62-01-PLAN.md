# Phase 62 Plan 01

## Objective

Define one canonical pluggable comparison contract and make both the EMEL embedding lane and the
existing C++ reference lane emit it without violating lane isolation.

## Tasks

1. Add a shared `embedding_compare/v1` record contract that carries timing, backend identity,
   fixture identity, output anchors, and optional binary output dumps.
2. Update the EMEL embedding benchmark runner to emit that contract in JSONL mode while preserving
   the maintained human-readable snapshot mode.
3. Update the C++ reference runner to emit the same contract cleanly in JSONL mode and stop
   mixing metadata chatter into the JSONL stream.

## Verification

- `cmake --build build/bench_tools_ninja --parallel --target embedding_generator_bench_runner`
- `python3 tools/bench/embedding_compare.py --reference-backend te_python_goldens --emel-runner build/bench_tools_ninja/embedding_generator_bench_runner --output-dir build/embedding_compare/te_python_goldens_all`

## Done Condition

The phase is complete when EMEL and reference lanes can emit the same machine-readable comparison
record format without sharing runtime objects or changing the maintained EMEL compute path.
