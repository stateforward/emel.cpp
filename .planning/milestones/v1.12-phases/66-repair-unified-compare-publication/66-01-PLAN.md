## Objective

Repair the unified compare publication path so maintained multi-result reference backends publish
complete machine-readable summaries and truthful operator-visible output.

## Tasks

1. Add a focused regression test that reproduces the dropped-record bug when two reference records
   share the same `compare_group`.
2. Update `tools/bench/embedding_compare.py` so summary generation and text output preserve every
   emitted maintained reference result instead of selecting only the first matching record.
3. Rerun the focused compare tests and the maintained `liquid_cpp` compare workflow to publish
   repaired evidence for the milestone closeout chain.

## Verification

- `ctest --test-dir build/bench_tools_ninja --output-on-failure -R '^embedding_compare_tests$'`
- `python3 tools/bench/embedding_compare.py --reference-backend liquid_cpp --emel-runner build/bench_tools_ninja/embedding_generator_bench_runner --output-dir build/embedding_compare/liquid_cpp_compare_text_repaired --case-filter text`

## Done Condition

The compare summary and stdout output preserve every emitted `liquid_cpp` baseline record for the
maintained text compare run, and focused bench-tool regression coverage proves the bug stays fixed.
