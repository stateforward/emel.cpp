# Phase 65 Plan 01

## Objective

Publish one maintained compare workflow that can run EMEL against pluggable Python or C++ backends
and emit reproducible compare artifacts plus focused verification coverage.

## Tasks

1. Add the unified compare driver that runs EMEL and the selected manifest backend, stores raw
   JSONL/vector artifacts, and writes a machine-readable compare summary.
2. Add the operator wrapper script for the common compare workflow.
3. Add focused bench-tool tests and verify the workflow on both the Python parity backend and the
   C++ baseline backend.

## Verification

- `ctest --test-dir build/bench_tools_ninja --output-on-failure -R '^embedding_compare_tests$'`
- `scripts/bench_embedding_compare.sh --reference-backend te_python_goldens --case-filter text --skip-emel-build --output-dir build/embedding_compare/wrapper_python_text`
- `python3 tools/bench/embedding_compare.py --reference-backend te_python_goldens --emel-runner build/bench_tools_ninja/embedding_generator_bench_runner --output-dir build/embedding_compare/te_python_goldens_all`
- `python3 tools/bench/embedding_compare.py --reference-backend liquid_cpp --emel-runner build/bench_tools_ninja/embedding_generator_bench_runner --output-dir build/embedding_compare/liquid_cpp_compare_text --case-filter text`
- `scripts/quality_gates.sh`

## Done Condition

The phase is complete when one consistent workflow can compare EMEL against either backend family,
the resulting artifacts publish backend and fixture identity plus similarity/baseline evidence, and
the maintained bench-tool tests plus full repo gate pass.
