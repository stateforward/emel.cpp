# Phase 83.2 Plan: Sortformer Modules Binding And Speaker Cache

## Goal

Land the maintained `mods.*` tensor contract, module projection/logit kernel surface, and fixed
speaker-cache state without claiming full Sortformer execution.

<rule_constraints>

- Cache state must be preallocated and fixed-profile.
- Numeric helpers must be allocation-free during dispatch.
- Behavior choice belongs to future SML guards/transitions, not module detail helpers.
- Do not update snapshots.

</rule_constraints>

## Tasks

1. Add shared `src/emel/diarization/sortformer/detail.hpp/.cpp` for reusable dense numeric work.
2. Move encoder dense helper usage to the shared Sortformer helper.
3. Add `src/emel/diarization/sortformer/modules/detail.hpp/.cpp` with:
   - maintained `mods.*` tensor binding
   - fixed-shape validation
   - projection and speaker-logit primitive kernels
4. Add `src/emel/diarization/sortformer/cache/detail.hpp` with fixed-profile cache state helpers.
5. Add focused tests for module binding, missing/shape rejection, cache reset/write/read, and
   deterministic module kernels.
6. Run focused build and diarization tests.
