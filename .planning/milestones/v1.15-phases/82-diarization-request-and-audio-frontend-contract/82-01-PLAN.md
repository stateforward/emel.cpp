# Plan 82-01: Diarization Request And Audio Frontend Contract

<goal>
Create the first diarization-owned SML actor for the maintained Sortformer profile. The actor
validates mono 16 kHz `float32` PCM requests, rejects invalid shape/capacity explicitly, and writes
a deterministic caller-owned feature matrix without running Sortformer tensors.
</goal>

<rule_constraints>
- Use Boost.SML orchestration with destination-first transition rows and explicit unexpected-event
  handling.
- Keep runtime request choices in `guards.hpp`; actions must only execute already-chosen work.
- Do not add Sortformer-specific fields to generic `model::data`.
- Do not add tool-only or external runtime fallbacks.
- Do not allocate during dispatch; all output storage is caller-owned.
</rule_constraints>

<tasks>

## Task 1: Add Diarization Request Component

Create `src/emel/diarization/request/` with canonical component files:

- `errors.hpp`
- `events.hpp`
- `context.hpp`
- `guards.hpp`
- `actions.hpp`
- `detail.hpp`
- `sm.hpp`

The request event should carry:

- required `model::sortformer::detail::execution_contract`
- required mono PCM span
- `sample_rate`
- `channel_count`
- caller-owned feature output span
- required `frame_count_out` and `feature_bin_count_out`
- optional error output and same-RTC callbacks

The actor should expose `emel::diarization::request::sm` and additive alias
`emel::DiarizationRequest`.

Verification: `cmake --build build/zig --target emel -j 4`.

## Task 2: Implement Explicit Validation And Frontend Preparation

Use explicit guards/transitions for:

- valid request
- invalid model/profile
- invalid sample rate
- invalid channel count
- invalid PCM shape
- insufficient output capacity
- successful feature extraction
- feature extraction failure

The feature-extractor kernel should deterministically compute a bounded feature matrix from the
already-validated PCM into caller-owned storage. It may use data-plane loops for frame/bin work but
must not choose behavior or allocate.

Verification: focused doctest cases for each valid and invalid path.

## Task 3: Add Focused Tests And Wire Build

Add `tests/diarization/request/lifecycle_tests.cpp` and register it in `CMakeLists.txt` with a
dedicated `emel_tests_diarization` shard. Tests must prove:

- valid mono 16 kHz PCM writes the maintained frame/bin shape
- output values are deterministic and finite
- invalid sample rate rejects explicitly
- invalid channel count rejects explicitly
- empty or wrong-sized PCM rejects explicitly
- insufficient feature output capacity rejects explicitly
- callbacks and error output are same-RTC and deterministic

Verification:

- `cmake --build build/zig --target emel_tests_bin -j 8`
- `ctest --test-dir build/zig --output-on-failure -R '^emel_tests_diarization$' -j 1`
- `EMEL_COVERAGE_CHANGED_ONLY=1 scripts/test_with_coverage.sh`

</tasks>
