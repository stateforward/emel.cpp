# Phase 92.5 Context

The milestone was reopened after Phase `92.4` because the Sortformer parity benchmark needs an
independent ONNX Sortformer reference. The recorded maintained baseline was produced by EMEL, so it
is a regression snapshot rather than a correctness oracle.

## Local Repo State

- No ONNX model artifact is checked in under `tests/models/`.
- No ONNX Runtime dependency is installed in the default Python environment.
- The current diarization compare driver, `tools/bench/diarization_compare.py`, compares one EMEL
  lane against one `reference` lane and defaults that reference to the self-recorded maintained
  regression snapshot from `bench_runner --mode=reference`.
- `scripts/bench_diarization_compare.sh` currently documents and runs the self-recorded regression
  lane only.

## Implementation Implications

- The ONNX model does not expose a raw-audio input. The reference lane must either reuse the
  maintained EMEL feature tensor as an explicitly labeled reference input contract or add a
  separate ONNX-side feature-preparation path and prove it matches the maintained feature contract.
- The ONNX output includes cache-prefixed logits. The phase must define which logits are comparable
  to EMEL's maintained `188 x 4` probability/segment output and must document any bounded drift.
- The ONNX lane must hard-fail when the runtime or model is missing; it must never substitute the
  self-recorded regression snapshot or EMEL output.
- The EMEL lane must remain isolated from ONNX-created state, tensors, caches, or formatter logic.
