# Phase 89 Plan 1: Maintained Sortformer E2E Runtime Orchestrator

## Goal

Add one maintained EMEL-owned Sortformer E2E actor that wires mono 16 kHz PCM to deterministic
speaker probabilities and bounded speaker segments through existing native request, encoder,
executor, and output components.

<rule_constraints>

- Use Boost.SML for orchestration and destination-first transition rows.
- Keep dispatch run-to-completion, no queues, no deferred work, and no dynamic allocation during
  dispatch.
- Keep runtime behavior decisions in guards/states/transitions.
- Do not add tool-only compute, fixed probability fixtures, synthetic projections, or external
  runtime/reference dependencies.
- Keep numeric work in existing component-owned Sortformer detail helpers.

</rule_constraints>

## Tasks

1. Add `src/emel/diarization/sortformer/pipeline/` with canonical `context.hpp`, `events.hpp`,
   `errors.hpp`, `guards.hpp`, `actions.hpp`, and `sm.hpp`.
2. Implement an `event::run` API using required references/spans for contract, PCM, output
   probabilities, output segments, output counts, and error output.
3. Model validation and phase progress in SML:
   model/profile input checks, probability capacity, segment capacity, encoder/modules contract
   readiness, request feature extraction, encoder-frame derivation, executor dispatch,
   probability computation, segment decoding, and publish success/error.
4. Add `emel::SortformerPipeline` as an additive top-level alias.
5. Add focused lifecycle tests proving deterministic PCM-to-segment output, repeated byte-stable
   records, invalid sample-rate rejection, and output-capacity rejection.
6. Wire the new test file into `CMakeLists.txt` and verify with the focused diarization shard.
