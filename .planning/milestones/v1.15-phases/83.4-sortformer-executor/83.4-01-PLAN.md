# Phase 83.4 Plan: Sortformer Executor Actor

## Goal

Add a diarization-owned Sortformer `executor` actor that composes the maintained stage surfaces
without introducing a generic runtime or misnaming the component as a generator or segmenter.

## Boundary

This phase starts at validated `188 x 512` encoder-frame input and executes:

- `mods.ep.*` encoder-frame projection
- fixed speaker-cache writes
- maintained `te.*` transformer layers
- deterministic `188 x 192` hidden-state handoff for Phase 84

It does not claim the full raw-PCM Conformer encoder path is complete. That remains a follow-on
phase because Phase 83.1 landed encoder tensor binding and primitive kernels, not the full
feature-to-encoder-frame computation.

## Tasks

1. Add `src/emel/diarization/sortformer/executor/` with canonical SML component files.
2. Model model-contract, tensor-contract, input-shape, output-capacity, execution-result, and
   publish decisions as explicit SML states/guards/transitions.
3. Reuse stage-owned module, cache, and transformer helpers.
4. Expose `emel::SortformerExecutor` from `src/emel/machines.hpp`.
5. Add focused lifecycle tests for deterministic output and explicit error routing.
6. Run focused diarization tests and full quality gates.
