# Phase 93 Plan 1: Recursive Sortformer ONNX Single-Thread Profile And Optimization

## Goal

Profile and optimize the maintained EMEL Sortformer runtime until it beats the ONNX Runtime CPU
single-thread benchmark reference on the pinned AMI fixture while preserving exact PyTorch/NeMo and
ONNX parity.

## Tasks

1. Confirm the ONNX benchmark lane is CPU-only and single-threaded.
2. Run a clean EMEL-vs-ONNX generated compare after restoring the last non-regressing candidate.
3. Run the strict PyTorch+ONNX generated compare before declaring the phase closed.
4. Keep only source changes that preserve exact output parity and improve or support benchmark
   scrutiny.
5. Record the final generated artifacts, hotspot outcome, and rejected optimization candidates.
6. Refresh roadmap/state/docs evidence from the generated records.

## Acceptance Criteria

1. ONNX records include source-backed CPU single-thread execution evidence.
2. EMEL beats ONNX single-thread on steady-state `ns_per_op`.
3. EMEL exact-matches PyTorch/NeMo and ONNX on output bytes, dimension, and checksum.
4. Any rejected optimization candidate is documented as rejected, not silently retained.
5. Phase `93` artifacts identify the exact generated record directory used for closeout.
