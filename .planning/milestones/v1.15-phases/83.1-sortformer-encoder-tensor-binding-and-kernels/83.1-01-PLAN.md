# Phase 83.1 Plan: Sortformer Encoder Tensor Binding And Kernel Surface

## Goal

Land the maintained encoder contract and first native encoder kernel surface for the fixed
Sortformer GGUF profile without claiming the whole Sortformer runtime is complete.

<rule_constraints>

- Runtime behavior selection must remain out of actions/detail helpers.
- Numeric data-plane loops may live in component-owned `detail.cpp`.
- Do not add placeholder source files for modules/cache/transformer/output.
- Do not update benchmark or snapshot baselines.

</rule_constraints>

## Tasks

1. Add `src/emel/diarization/sortformer/encoder/detail.hpp/.cpp` with:
   - maintained encoder constants
   - pre-encoder tensor views
   - 17 layer tensor views
   - fixed-shape validation for required `enc.*` tensors
2. Add component-owned deterministic f32 kernel helpers for the first encoder surface:
   - affine normalization over a 512-wide frame vector
   - dense projection over fixed maintained dimensions
3. Add focused tests under `tests/diarization/sortformer/encoder/` covering:
   - valid maintained tensor binding
   - missing tensor rejection
   - shape mismatch rejection
   - deterministic kernel output
4. Wire new source/test files into CMake.
5. Run focused build and diarization tests.

## Verification

- `cmake --build build/coverage --target emel_tests_bin -j 8`
- `ctest --test-dir build/coverage --output-on-failure -R '^emel_tests_diarization$' -j 1`

## Completion Criteria

Phase 83.1 is complete when the maintained `enc.*` contract is bound and validated in EMEL-owned
`src/`, and the first encoder kernel surface is deterministic and tested. Full modules/cache,
transformer execution, and end-to-end diarization output remain later phases.
