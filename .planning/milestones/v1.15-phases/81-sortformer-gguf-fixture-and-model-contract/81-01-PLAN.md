# Plan 81-01: Sortformer GGUF Fixture And Model Contract

<goal>
Pin the maintained Sortformer GGUF fixture contract and add explicit `sortformer` architecture
acceptance/rejection behavior.
</goal>

<rule_constraints>
- Register `sortformer` as a first-class model architecture.
- Keep metadata binding in `src/emel/model/sortformer/`.
- Require deterministic rejection for malformed or off-scope Sortformer contracts.
- Use synthetic loader tests; do not download or commit the full model artifact.
</rule_constraints>

## Tasks

1. Add `src/emel/model/sortformer/detail.hpp` and `detail.cpp` with:
   - `execution_contract`
   - GGUF metadata binding for `sortformer.*` keys
   - required tensor-family validation
   - maintained streaming profile defaults
2. Register `sortformer` in the model architecture registry and expose
   `is_sortformer_execution_architecture`.
3. Add Sortformer metadata storage to `emel::model::data`.
4. Document the maintained fixture in `tests/models/README.md`.
5. Add focused model-loader lifecycle tests for:
   - GGUF hparam binding
   - supported architecture predicate
   - execution contract success
   - deterministic rejection of invalid metadata/tensor families
6. Run focused tests, then the required quality gate if feasible.

## Verification

- `emel_tests` includes Sortformer loader/model-contract tests.
- `git diff --check` passes.
- `scripts/quality_gates.sh` is run after implementation, unless blocked by an environment/tool
  failure that is reported.
