# Phase 92.1 Plan 1: Maintained GGUF Contract Parity And Benchmark Truth Repair

## Goal

Replace the scaffolded Sortformer parity and benchmark fixture path with the real maintained GGUF
loader-built execution contract and one pinned real-audio fixture on top of the repaired
maintained runtime from Phase `92.2`, so `v1.15` publishes a truthful steady-state ARM baseline
for the maintained EMEL diarization lane.

<rule_constraints>
- Do not let parity or benchmark use `tools/bench/diarization/sortformer_fixture.hpp`
  `make_contract(...)` or synthetic generated PCM as the maintained proof path.
- Keep the EMEL lane entirely EMEL-owned: real maintained GGUF artifact, EMEL loader, EMEL
  contract builder, and `sortformer/pipeline::sm`.
- Keep one-time costs such as model loading, fixture decode, and setup outside the primary
  optimization claim. If reported, publish them as secondary prepare/setup evidence.
- Keep the reference lane isolated and non-authoritative for maintained-path truth.
- Do not invent an external ARM target when none is source-backed; establish the first truthful
  maintained baseline on exact hardware instead.
- Do not update snapshots without explicit user consent.
</rule_constraints>

## Tasks

1. Land a maintained Sortformer fixture path.
   - Add a maintained fixture helper that loads
     `tests/models/diar_streaming_sortformer_4spk-v2.1.gguf` through the real model loader flow.
   - Build the execution contract via `src/emel/model/sortformer/detail.cpp` rather than a
     benchmark-local fabricated contract.
   - Reuse only pipeline-driving helpers from the old benchmark fixture; remove maintained-path
     dependence on the synthetic model/PCM scaffold.

2. Pin one real-audio maintained fixture with provenance.
   - Add one maintained public multi-speaker audio clip or repo-native derived fixture with clear
     source, license, and conversion metadata.
   - Prefer an upstream/model-card-aligned example if a concrete public clip exists; otherwise use
     a documented public substitute.
   - Keep the benchmark input contract fixed to mono `16 kHz` audio and document any conversion.

3. Switch parity to the maintained path.
   - Update `tests/diarization/sortformer/parity/lifecycle_tests.cpp` to use the maintained model
     fixture and real-audio fixture.
   - Record deterministic maintained output expectations for the pinned clip and verify the EMEL
     lane through `sortformer::pipeline::sm`.
   - Add a regression that fails if the maintained parity lane regresses to a fabricated contract
     or synthetic PCM path.

4. Switch benchmark publication to the maintained path.
   - Update `tools/bench/diarization/sortformer_bench.cpp` to preload the maintained model and
     audio fixture, measure steady-state pipeline runtime as the primary metric, and report
     prepare/setup costs separately.
   - Preserve current compare/report metadata shape where practical, but make the maintained-path
     provenance explicit in emitted notes.
   - Keep the reference lane separate and avoid sharing model/runtime/cache objects.

5. Repair docs and milestone evidence.
   - Update `docs/templates/benchmarks.md.j2` and `docs/benchmarks.md` so Sortformer claims refer
     to the maintained GGUF loader-built contract and pinned real-audio fixture.
   - Add `92.1-01-SUMMARY.md`, `92.1-VERIFICATION.md`, and update milestone state/audit evidence so
     archival depends on this maintained-path repair.
