# Phase 92.2 Plan 1: Maintained Sortformer Runtime Contract Repair

## Goal

Repair the maintained Sortformer `src/` runtime so it accepts and executes the real GGUF contract:
maintained preprocessor inputs, maintained FastConformer/encoder contract, maintained
projection/transformer path, and maintained sigmoid speaker head.

<rule_constraints>
- Do not reintroduce synthetic `enc.pre.*` or synthetic PCM/features as a compatibility shim.
- Keep runtime work in `src/`; do not hide source gaps in `tools/bench` or parity-only helpers.
- Preserve EMEL lane isolation from external runtimes.
- Keep one-time binding/search work outside hot dispatch paths.
- Do not update snapshots without explicit user consent.
</rule_constraints>

## Tasks

1. Repair maintained tensor interpretation.
   - Update Sortformer binder shape checks so they match the maintained GGUF tensor dimension
     ordering for encoder layer tensors, module tensors, and transformer tensors.
   - Add or update focused tests that cover the maintained dimension ordering explicitly.

2. Repair the maintained speaker head path.
   - Update the output path to match the upstream inference head:
     ReLU -> `first_hidden_to_hidden` -> ReLU -> `single_hidden_to_spks` -> sigmoid.
   - Keep inference deterministic and allocation-free.

3. Replace the synthetic frontend contract.
   - Derive maintained acoustic features from the real `prep.feat.fb` / `prep.feat.win`
     contract instead of the current synthetic request feature extractor.
   - Rework the encoder entry path so the maintained pipeline consumes the actual
     `enc.pre.*` `dw_striding` stack on truthful `1504 x 128` feature frames and produces
     truthful `188 x 512` embeddings for the remaining encoder.
   - Remove the current synthetic `188 x 128` feature shortcut and any logic that treats the
     pre-encode stage as a fabricated compatibility path.

4. Reprove maintained runtime acceptance.
   - Drive the pinned maintained GGUF and pinned AMI clip through the maintained pipeline actor.
   - Confirm the live runtime no longer fails `pipeline::error::tensor_contract`.
   - Record the first truthful maintained output checksum and non-zero segment count once the
     real frontend path runs.

5. Refresh runtime-focused phase evidence.
   - Write `92.2-01-SUMMARY.md` and `92.2-VERIFICATION.md` with the repaired contract path,
     exact runtime changes, and maintained execution evidence.
   - Hand Phase `92.1` a truthful runtime to use for parity/benchmark publication.
