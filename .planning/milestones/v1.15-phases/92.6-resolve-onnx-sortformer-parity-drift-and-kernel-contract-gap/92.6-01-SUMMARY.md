# Phase 92.6 Plan 1 Summary: Resolve Sortformer PyTorch Parity And ONNX Benchmark Drift

## Outcome

Phase `92.6` is complete. The correctness target is now source-backed by the official
PyTorch/NeMo model-card lane, with ONNX Runtime retained as the CPU benchmark reference lane after
it exact-matches PyTorch/NeMo.

The original Phase `92.5` drift was not a recorded-baseline issue. The self-recorded EMEL snapshot
reported `71` segments while both executable reference lanes reported `17` segments with checksum
`4249677247906920305`. Phase `92.6` made the lane roles explicit:

- `pytorch.nemo.sortformer.v2_1`: parity reference lane.
- `onnx.sortformer.v2_1`: benchmark reference lane.
- `recorded.diarization.baseline`: self-recorded regression snapshot only.

The maintained EMEL runtime now exact-matches both executable reference lanes on the pinned AMI
fixture at `output_dim=17` and checksum `4249677247906920305`.
