# Plan 139-01: Generator Benchmark Diagnostics Boundary Closure

<objective>
Replace context-reading generator `sm` diagnostic getters with a public generator diagnostics event,
cut benchmark/parity/test call sites over to that event, fix Phase 138 summary frontmatter, and
prove `TEXTGEN-07` is source-backed without actor-internal benchmark publication proof.
</objective>

<rule_constraints>
- All runtime behavior remains modeled in `src/emel/text/generator/sm.hpp` transitions.
- `event::capture_diagnostics` must carry required data by reference and stay small.
- Actions may copy current actor metrics from context; guards must not mutate context.
- `sm` member functions must not read or write `this->context_`.
- Benchmark and paritychecker code must not include or name text-generator `detail`, `action`, or
  `guard` internals for maintained generation proof.
- Verification must use source scans plus focused tests; do not accept artifact-only agreement.
</rule_constraints>

<tasks>

## Task 1: Add Public Diagnostics Event

Files:
- `src/emel/text/generator/events.hpp`
- `src/emel/text/generator/actions.hpp`
- `src/emel/text/generator/sm.hpp`

Steps:
1. Define a public diagnostics snapshot type containing existing generation counters and contract
   stage counts.
2. Define `event::capture_diagnostics` with a required diagnostics output reference.
3. Add an allocation-free action that copies diagnostics from actor context into the event output.
4. Add explicit transition rows for the public event in stable generator states.
5. Remove or replace the existing context-reading `sm` diagnostic methods.

4. Strengthen benchmark source scans so the whole maintained benchmark source cannot include or
