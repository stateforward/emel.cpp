# Phase 133 Plan: Generator SML Rule Repair

<rule_constraints>
- The public generator wrapper must not contain runtime branching that decides initialize success or
  failure.
- Guards in `guards.hpp` and explicit transitions in `sm.hpp` own readiness and invalid-request
  routing.
- Actions only publish the already-selected initialize outcome.
</rule_constraints>

## Tasks

1. Add a focused regression proving `sm::process_event(event::initialize)` contains no runtime
   branch before dispatching the internal initialize run event.
2. Remove the wrapper-level model/conditioner readiness branch from
   `src/emel/text/generator/sm.hpp`.
3. Let the existing `valid_initialize` / `invalid_initialize` transition rows route missing model
   or conditioner dependencies to `action::reject_initialize`.
4. Run the generator/runtime shard and rule-focused source checks.
