# Phase 144 Plan: Text Generator Runtime Route Ownership Closure

## Goal

Close the v1.17 audit gap where the maintained text generator runtime still made
route decisions from action-called `detail.hpp` helpers instead of explicit
Stateforward.SML guard/transition rows.

## Tasks

1. Split generator scalar matmul execution into explicit fixed routes:
   `packed_q8_0`, `q8_k`, `native_quantized`, and `kernel`.
2. Replace generic decode and scalar prefill kernel actions with route-named
   actions that bind a single selected `run_kernel_*` entrypoint.
3. Add guard-owned route predicates for materialized logits and preselected
   argmax, including the mixed native quantized route used by maintained
   quantized fixtures.
4. Update parent generator and prefill SML transition tables so runtime route
   selection is visible in `sm.hpp`.
5. Add source regressions proving generic prefill/decode wrappers and generic
   action bindings do not return.
6. Validate generator runtime behavior, source structure, paritychecker, domain
   boundaries, and the scoped quality gate.

## Verification Plan

- Build `emel_tests_bin`.
- Run `ctest --test-dir build/debug -R emel_tests_generator_and_runtime --output-on-failure`.
- Run source-pattern checks for the removed generic wrappers under
  `src/emel/text/generator`.
- Run `scripts/check_domain_boundaries.sh`.
- Run `git diff --check`.
- Run `ctest --test-dir build/paritychecker_zig -R paritychecker_tests --output-on-failure`.
- Run scoped `scripts/quality_gates.sh` with changed-file scoping and
  `EMEL_QUALITY_GATES_BENCH_SUITE=generation`.
