# Plan 152-01: Parity Behavior And Lane-Isolation Closure

<objective>
Close v1.18 by proving maintained parity behavior still passes and adding source regressions that
make lane-boundary violations visible before future parity tool refactors land.
</objective>

<tasks>
<task id="1" type="auto">
<title>Remove direct actor-helper bridge</title>
<instructions>
Replace the reference-side Plamo2 byte-token helper call into EMEL detokenizer action detail with a
local reference-lane parser, and drop unused actor detail includes.
</instructions>
<acceptance_criteria>
- `parity_engines.cpp` no longer includes `emel/text/detokenizer/actions.hpp`.
- `parity_engines.cpp` no longer includes `emel/text/jinja/parser/detail.hpp`.
- Reference output rendering remains behavior-compatible under existing generation parity tests.
</acceptance_criteria>
</task>

<task id="2" type="auto">
<title>Add source checks for lane isolation</title>
<instructions>
Extend paritychecker doctests so shared runner sources cannot include llama/ggml runtime state or
EMEL model/runtime state, and engine source checks prove tokenizer and generation lanes keep EMEL
and reference objects separate.
</instructions>
<acceptance_criteria>
- Tests fail if shared runner code starts owning lane runtime objects.
- Tests fail if tokenizer/generation source shows obvious EMEL/reference object reuse.
- Tests fail if the removed actor-helper bridge reappears.
</acceptance_criteria>
</task>

<task id="3" type="auto">
<title>Validate maintained behavior</title>
<instructions>
Run the focused paritychecker build, paritychecker test suite, executable build, and changed-file
scoped quality gate.
</instructions>
<acceptance_criteria>
- Existing tokenizer, GBNF, kernel, Jinja, and generation parity tests pass.
- No snapshot update is committed without explicit approval.
</acceptance_criteria>
</task>

<task id="4" type="auto">
<title>Close requirement traceability</title>
<instructions>
Update roadmap, requirements, state, summary, review, validation, and verification documents for
Phase 152.
</instructions>
<acceptance_criteria>
- `PARITY-03` and `LANE-02` are complete.
- Roadmap analysis sees all v1.18 phases complete.
</acceptance_criteria>
</task>
</tasks>
