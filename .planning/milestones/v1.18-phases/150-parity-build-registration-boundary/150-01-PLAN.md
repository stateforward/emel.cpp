# Plan 150-01: Parity Build Registration Boundary

<objective>
Make paritychecker runner/engine build wiring explicit and shared across the executable and tests,
with source-backed proof that engine registration is localized and fail-closed.
</objective>

<rule_constraints>
## Rule Traceability

- `BUILD-01` requires modular build boundaries. Tasks 1 and 2 factor CMake into explicit runner,
  registration, implementation, tokenizer, and reference-support source groups.
- `BUILD-02` requires auditable registration and no hidden fallback. Tasks 3 and 4 add source and
  runtime tests for adapter names and invalid-mode handling.
- `ENGINE-02` requires future engine additions to stay localized. Tasks 1 and 4 prove the localized
  CMake/source registration surface.
</rule_constraints>

<tasks>
<task id="1" type="auto">
<title>Factor paritychecker CMake source groups</title>
<read_first>
- `tools/paritychecker/CMakeLists.txt`
</read_first>
<instructions>
Replace duplicated executable/test source lists with explicit source groups for CLI, runner,
engine registration, engine implementation, tokenizer engines, and reference support. Compose a
single common source list used by both targets.
</instructions>
<acceptance_criteria>
- `paritychecker` and `paritychecker_tests` share `PARITYCHECKER_COMMON_SOURCES`.
- Future engine source additions have an obvious engine source group.
</acceptance_criteria>
</task>

<task id="2" type="auto">
<title>Keep registration fail-closed and auditable</title>
<read_first>
- `tools/paritychecker/parity_engine.cpp`
- `tools/paritychecker/parity_engine.hpp`
</read_first>
<instructions>
Keep the current explicit adapter metadata and invalid-mode `nullptr` behavior. Add only minimal
helpers if needed for tests; do not add a silent default engine fallback.
</instructions>
<acceptance_criteria>
- Invalid mode lookup remains `nullptr`.
- Engine names and mode mapping remain visible in a small source file.
</acceptance_criteria>
</task>

<task id="3" type="auto">
<title>Add build and registration source regressions</title>
<read_first>
- `tools/paritychecker/paritychecker_tests.cpp`
- `tools/paritychecker/CMakeLists.txt`
</read_first>
<instructions>
Add doctest source checks proving the modular CMake groups exist, both targets consume the shared
source list, engine implementation source is localized, and invalid mode does not fallback.
</instructions>
<acceptance_criteria>
- Tests fail if direct duplicated source lists replace the shared common source group.
- Tests fail if `parity_engine.cpp` adds a default tokenizer fallback for invalid modes.
</acceptance_criteria>
</task>

</task>
</tasks>
