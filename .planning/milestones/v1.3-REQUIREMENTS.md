# Requirements: EMEL

**Defined:** 2026-03-22
**Core Value:** Prove real end-to-end behavior with explicit SML orchestration and
parity-oriented verification before widening API surface or model scope.

## v1 Requirements

### ARM Flash Kernel

- [x] **PORT-01**: The canonical Llama-68M ARM generation slice can execute
  `op_flash_attn_ext` through an AArch64-specific optimized implementation instead of the shared
  scalar workspace path.
- [x] **PORT-02**: The AArch64 flash-attention implementation preserves zero-allocation hot-path
  behavior and uses backend-owned reusable workspace or buffers.
- [x] **PORT-03**: Unsupported or out-of-contract AArch64 flash requests publish explicit
  deterministic behavior and do not silently claim optimized-path execution.

### Architecture Contract

- [x] **ARCH-01**: The ARM flash optimization remains a data-plane replacement inside the existing
  generator -> graph -> processor -> kernel chain and does not require widening the public API
  surface or introducing queue-based orchestration.

### Verification

- [x] **PAR-03**: `tools/paritychecker --generation` proves that the maintained canonical ARM
  Llama-68M workload exercised the AArch64 optimized flash path.
- [x] **VER-02**: Tests cover optimized-path correctness, negative selection or unsupported-shape
  behavior, and the allocation-free contract for the AArch64 flash path.

### Benchmarking

- [x] **BENCH-04**: `tools/bench` can run the canonical ARM generation workload through the
  optimized AArch64 flash path on the maintained compare surface.
- [x] **BENCH-05**: Benchmark outputs publish enough proof or attribution to distinguish AArch64
  flash-path execution from surrounding generator/runtime cost on the maintained canonical
  workload.
- [x] **BENCH-06**: Canonical ARM benchmark results show a measurable latency improvement over the
  current shared-scalar ARM flash baseline on at least one maintained compare case.

## v2 Requirements

### Generator Math

- **GEN-03**: Optimize ARM generator-side RMSNorm, RoPE, residual-add, and SwiGLU math after the
  standalone AArch64 flash gain is measured.

### Broader Coverage

- **FLASH-03**: Broaden flash attention beyond the canonical Llama-68M shape and workload
  contract.
- **MODEL-01**: Roll optimized ARM flash attention out to additional model fixtures after the
  canonical path remains correct and benchmarked.

### Benchmark Policy

- **BENCH-07**: Revisit whether noisy benchmark drift should become a blocking repo gate once ARM
  compare evidence stabilizes.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Non-ARM backend flash specialization | This milestone is specifically about closing the ARM CPU gap first. |
| Generator-side ARM math pack outside flash | Keep the milestone narrow until the AArch64 flash gain is isolated and measured. |
| Broader model or workload rollout | The canonical Llama-68M slice remains the only accepted truth anchor in this milestone. |
| State-machine or orchestration rewrites | Current evidence points to scalar data-plane cost, not SML orchestration, and the milestone must preserve the architecture rules. |
