# Phase 217 Plan 01

## Goal

Make read/copy behavior and scope boundaries source-backed through public-dispatch
tests and regression guardrails.

<rule_constraints>

- Public behavior tests must drive actors with `process_event(...)` and inspect
  state through `is(...)` or `visit_current_states(...)`.
- Source guardrails must fail if `model/loader` or maintained tools bypass
  `io/loader` and call direct `io/read` events.
- Source guardrails must prove tensor residency remains in `model/tensor`.
- The v1.25 read/copy path must not use `staged_read` naming that implies deferred
  staged/chunked constrained-memory behavior.

</rule_constraints>

## Tasks

### Task 1 - Clarify public strategy naming

Rename the copy strategy from `staged_read` to `read_copy` across `io/loader`,
tests, maintained tool helper code, and generated architecture docs.

Acceptance:
- Active source, tests, tool helper, and generated architecture docs contain no
  `staged_read` or `strategy_staged` markers.
- Maintained tool environment selection accepts `read` and `read_copy`.

### Task 2 - Add behavior and naming guardrails

Add doctest guardrails that prove `io/loader` routes `read_copy` through the read
actor and keeps the old staged-policy wording out of active runtime/tool sources.

Acceptance:
- `tests/io/loader/lifecycle_tests.cpp` validates `read_copy` success and failure
  behavior through public dispatch and ready-state inspection.
- The guardrail test constructs forbidden staged-policy strings dynamically and
  checks active runtime/tool sources.

### Task 3 - Add ownership and reach-through guardrails

Add source guardrails that prove `model/tensor` owns read/copy residency and that
`model/loader` plus maintained tools do not use direct `io/read` event plumbing.

Acceptance:
- Tensor guardrail checks `effect_commit_request_read_load` commits
  `event::lifecycle::resident` in `model/tensor`.
- `io/read` source remains free of `model/tensor` residency references.
- Maintained tools and `model/loader` do not include direct `io/read` event types
  or `read_tensor_request` scaffolding.
