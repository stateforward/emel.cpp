# Phase 220 Plan 01

## Goal

Repair the tensor read/copy outcome graph so final tensor outcomes are selected
by explicit guarded transitions over typed same-RTC read result data, not by
callback-mutated tensor status.

<rule_constraints>

- Runtime behavior choice belongs in `sm.hpp` guarded by predicates from
  `guards.hpp`.
- Tensor read success/error selection must not depend on callbacks that mutate
  tensor-local status fields.
- Actions may dispatch the already-selected `io/read` path and copy resulting
  payload fields, but must not branch or select the final outcome.
- Tests must exercise public `process_event(...)` surfaces and SML state
  inspection.

</rule_constraints>

## Tasks

### Task 1 - Add typed same-RTC read result capture

Extend `io/read` with a result carrier and overload that captures accepted,
success, error category, bytes copied, and target buffer from the same RTC
dispatch without requiring public callbacks to decide the caller's next path.

Acceptance:
- `src/emel/io/read/events.hpp` defines the typed result carrier.
- `src/emel/io/read/sm.hpp` captures result data while preserving existing
  public callback behavior.
- `tests/io/read/lifecycle_tests.cpp` proves result capture succeeds without
  user-provided callbacks.

### Task 2 - Remove callback-mediated tensor read selection

Update `model/tensor` read dispatch to use the typed `io/read` result carrier
and explicit guards over that carrier.

Acceptance:
- `src/emel/model/tensor/detail.hpp` no longer has `io_read_ok` or
  `io_read_err` mirrored status fields.
- `src/emel/model/tensor/actions.hpp` no longer defines read callback thunks for
  tensor outcome selection.
- `src/emel/model/tensor/guards.hpp` routes success/error categories from the
  typed read result.

### Task 3 - Prove representative outcomes

Update tensor doctests/source guardrails for success, unsupported, validation
failure, file open failure, and file read failure through public dispatch.

Acceptance:
- `tests/model/tensor/lifecycle_tests.cpp` proves representative read outcomes.
- Guardrails reject callback-mediated tensor read outcome selection.
- Focused build/tests and scoped quality gate pass before marking Phase 220
  complete.
