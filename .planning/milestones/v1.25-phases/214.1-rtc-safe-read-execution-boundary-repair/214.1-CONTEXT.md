# Phase 214.1 Context

The milestone audit found that the live `src/emel/io/read` source no longer matches the
Phase 214 artifacts. The implementation now uses caller-provided source spans and
`source_error` instead of dispatch-time platform open/seek/read/close calls.

This phase closes only that audit gap. It must prove the read actor remains RTC-safe,
allocation-free, and source-backed while preserving copied-byte success, deterministic
errors, and caller-owned target buffer semantics. It does not claim tensor integration,
maintained benchmark/parity coverage, public runtime reporting, or closeout publication.

## Source Truth

- `src/emel/io/read/actions.hpp` copies from `source_buffer + file_offset` into
  `target_buffer` and publishes `events::read_tensor_done`.
- `src/emel/io/read/guards.hpp` models source open/seek/read/short-read outcomes through
  guards over immutable event payload.
- `src/emel/io/read/context.hpp` remains empty.
- `CMakeLists.txt` does not build `src/emel/io/read/actions.cpp`; the file is deleted.
- `tests/io/read/lifecycle_tests.cpp` drives the actor through public
  `process_event(...)` and verifies ready-state recovery.

## Non-Goals

- Do not add low-level filesystem calls to SML guards, actions, detail helpers, or
  transition helpers.
- Do not move residency ownership from `model/tensor`.
- Do not update benchmark/parity/model publication claims in this phase.
