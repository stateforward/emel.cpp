# Phase 219 Plan 01

## Goal

Make maintained benchmark, paritychecker, and embedded probe lanes source-backed
by a maintained `src/emel/io/read` source contract, with guardrails that fail if
tool-local full-file readers feed read/copy evidence again.

<rule_constraints>

- Filesystem reads must not occur inside SML guards/actions or dispatch-critical
  code.
- Tool setup code must not define duplicate low-level `read_file_bytes` helpers
  for maintained read/copy evidence lanes.
- Read/copy evidence must be emitted only through public model-loader/I/O
  surfaces after the maintained source contract provides source bytes.
- Unsupported or missing read strategy behavior must remain reported as
  unsupported, not as read/copy parity or performance evidence.

</rule_constraints>

## Tasks

### Task 1 - Add maintained source-byte contract

Superseded by Phase 222: use public `src/emel/io/source/any.hpp` for
setup-time full-file source loading. The helper returns `io/read` error
categories and is not called from actor guards/actions.

Acceptance:
- `src/emel/io/source/any.hpp` exposes a source-file byte loading helper.
- `src/emel/io/read/sm.hpp`, `actions.hpp`, and guards remain free of
  dispatch-time filesystem reads.

### Task 2 - Wire maintained tools to the source contract

Replace tool-local `read_file_bytes` usage in:

- `tools/bench/generation_bench.cpp`
- `tools/bench/diarization/sortformer_fixture.hpp`
- `tools/embedded_size/emel_probe/main.cpp`
- `tools/paritychecker/parity_engines.cpp`
- `tools/paritychecker/parity_assets.hpp`
- `tools/paritychecker/parity_assets.cpp`

Acceptance:
- Maintained tool lanes no longer define local `read_file_bytes` helpers.
- EMEL read/copy lanes still bind/report strategy through public
  `model/loader` and I/O surfaces.

### Task 3 - Add guardrails and focused verification

Update tests so source scans fail on tool-local read substitutes and prove the
maintained helper is used by maintained lanes.

Acceptance:
- `tests/model/loader/lifecycle_tests.cpp` rejects `read_file_bytes` in
  maintained tool sources and requires public `emel::io::source::load_file_bytes`.
- `tools/paritychecker/paritychecker_tests.cpp` uses the maintained helper for
  common source loading proof.
- Focused build/tests and scoped quality gate pass before marking Phase 219
  complete.
