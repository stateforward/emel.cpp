# Phase 212 Plan 01: Read Strategy Component Boundary

## Goal

Create the `src/emel/io/read` Stateforward.SML component skeleton and ownership surface
so future phases can land read validation + platform gating (Phase 213), real read
execution + error taxonomy + transient-resource lifetime (Phase 214), tensor-side
integration (Phase 215), public runtime/evidence exposure (Phase 216), behavior tests
+ scope guardrails (Phase 217), and milestone publication (Phase 218) without
re-creating the boundary.

### Task 6 — Boundary doctest

- Path: `tests/io/read/lifecycle_tests.cpp`.
- Drive the actor exclusively through `process_event(...)` and inspect via
  `is(state<...>)` / `visit_current_states(...)`. Do NOT call internal helpers.
- Doctest cases:
  1. "io read exposes canonical machine aliases at component boundary" — both
     `emel::io::read::sm` and `emel::IoRead` instantiate.
  2. "io read fails closed for unsupported platforms at the boundary" — accepted
     request trigger lands in `state_unsupported_platform_error_decision` and fires
     the error callback with `error::unsupported_platform`.
  3. "io read fails closed without an error callback" — same as above with the
     callback absent; actor returns to `state_ready` deterministically.
  4. "io read recovers to ready after fail-closed dispatches" — sequential
     unsupported-platform dispatches end in `state_ready`.
  5. "io read reports state_ready via visit_current_states" — `visit_current_states`
     reports only `state_ready` after a fail-closed dispatch.
  6. "io read handles unexpected events deterministically" — ready state accepts an
     event of an unrelated type and remain in their current state without firing
     callbacks.
  7. "io read boundary contains no concrete platform read calls" — read each
     `src/emel/io/read/*.hpp` file content and assert the source text contains no
     `pread`, `read(`, `lseek`, `open(`, `close(`, `ReadFile`, `CreateFileW`,
