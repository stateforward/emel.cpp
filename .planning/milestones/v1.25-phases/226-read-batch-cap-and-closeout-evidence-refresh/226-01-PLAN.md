# Phase 226 Plan 01: Read Batch Cap And Closeout Evidence Refresh

<rule_constraints>
## SML And Runtime Rules

- Keep runtime behavior selection in `sm.hpp` transitions using guards from
  `guards.hpp`.
- Do not add runtime branching to actions or action-called helpers.
- Keep dispatch run-to-completion and allocation-free.
- Drive behavior through public `process_event(...)` tests and SML state inspection.
</rule_constraints>

## Goal

Close the two nonblocking tech-debt items from the refreshed v1.25 milestone audit:

1. Public `io/read::event::read_tensor_batch` needs an independent hard cap before
   per-span iteration or copy.
2. Closeout evidence needs to distinguish historical dyld fallback notes from current
   direct focused CTest evidence.

## Tasks

1. Add public read batch count contract.
   - Add a read-owned `k_max_read_batch_tensors` constant alongside existing `io/read`
     limits.
   - Add explicit count-valid/count-invalid guards.
   - Add an explicit SML transition leg so over-large batches fail before any per-span
     validation loop or copy loop.

2. Add public-dispatch doctests.
   - Prove exact-cap batches are accepted and copied.
   - Prove over-cap batches publish `invalid_request`, do not publish done, and return to
     `state_ready`.

3. Refresh evidence artifacts.
   - Write `226-SUMMARY.md`, `226-VERIFICATION.md`, and `226-VALIDATION.md`.
   - Refresh `.planning/v1.25-MILESTONE-AUDIT.md` after validation.
   - Update active and archived roadmap/state progress.

4. Run gates.
   - Focused `emel_tests_io`.
   - Domain-boundary checks if source ownership is touched.
   - Changed-file scoped `scripts/quality_gates.sh`.
   - Maintained snapshots, benchmarks, or model artifacts only if a maintained command
     reports required updates.

## Success Criteria

- Public `read_tensor_batch` dispatch rejects over-large spans before per-span loops.
- Exact cap and over-cap behavior are covered through public dispatch doctests.
- Closeout evidence records current command results and historical dyld fallback status
  truthfully.
- Quality gates pass without benchmark-regression override.
