# Phase 215 Plan 01

## Goal

Add a tensor-owned read/copy load request path that dispatches through public `emel/io`
events and represents read success and failure outcomes with explicit tensor states and
public `_done`/`_error` events.

## Tasks

### Task 1 - Add tensor read request contracts

Add `request_read_load`, `request_read_load_done`, and `request_read_load_error` to
`model/tensor/events.hpp`. Add read-specific tensor error categories and runtime
carriers.

Acceptance:
- Public tensor events carry read source-span inputs and callback outputs.
- Read-specific tensor errors preserve the upstream `io/read` error for diagnostics.

### Task 2 - Wire tensor actor to io/read

Inject `emel::io::read::sm` into tensor context and add read load transitions in
`model/tensor/sm.hpp`.

Acceptance:
- Runtime behavior choice lives in guards and transition rows.
- Success commits only the caller-owned target buffer to tensor residency.
- Unsupported actor, invalid request, already resident, read validation failure, file
  open failure, and file read failure have explicit states, with remaining upstream
  read errors preserved in the public error payload.

### Task 3 - Prove public dispatch behavior

Add `tests/model/tensor/lifecycle_tests.cpp` coverage for read success, missing read
actor, invalid request, file-open failure, and file-read failure through
`process_event(...)` and state inspection.

Acceptance:
- Model/tensor lifecycle doctests pass.
- `ctest -R emel_tests_model_and_batch` and `ctest -R emel_tests_io` pass.
- Generated architecture docs reflect the new tensor read transition graph.
