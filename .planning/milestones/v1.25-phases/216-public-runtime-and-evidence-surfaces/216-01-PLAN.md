# Phase 216 Plan 01

## Goal

Expose truthful model-load strategy selection and reporting through public model-loader,
tensor, and I/O runtime contracts, and remove maintained tool-lane reach-through into
actor internals.

<rule_constraints>

- Keep `model/loader` orchestration-only; no low-level read/copy code belongs there.
- Runtime behavior choice must remain in SML guards/transitions or public tool setup,
  not in actions that choose hidden behavior paths.
- Maintained tools must not call `process_event(...)` inside load callbacks to inspect
  actor internals.
- Benchmark and parity evidence may report read/copy only from public model-loader
  done/error events produced by the actual EMEL lane.

</rule_constraints>

## Tasks

### Task 1 - Add public model-loader strategy evidence

Extend model-loader done/error events so public callbacks distinguish the requested
I/O strategy from the strategy actually used.

Acceptance:
- Successful read/copy flow reports `used_io_strategy == read_copy`.
- Unsupported/fail-closed strategy errors report the requested strategy and
  `used_io_strategy == none`.
- Existing mmap/non-I/O fields remain available and truthful.

### Task 2 - Wire maintained tools through public load strategy binding

Add a shared maintained tool helper that selects a model-load I/O strategy from
`EMEL_MODEL_LOAD_IO_STRATEGY` and binds it to public `model::loader::event::load`
plus an injected `io::loader::sm`.

Acceptance:
- Generation benchmark, Sortformer benchmark fixture, embedded probe, and
  paritychecker all use the helper.
- Tool load callbacks consume `ev.used_io_strategy` and `ev.requested_io_strategy`.
- Tool load callbacks do not dispatch actor probes to infer strategy usage.

### Task 3 - Prove maintained evidence surfaces

Add source-backed tests and compile the affected maintained tool targets.

Acceptance:
- Model-loader doctests cover read/copy success and unsupported strategy error
  evidence.
- Source guard checks prove the four maintained tool lanes use the public helper and
  no longer contain the former callback-time tensor capture dispatch.
- Generation, diarization, paritychecker, and embedded probe tool targets compile.
