# Phase 222 Plan 01

## Goal

Repair maintained read/copy source loading so benchmark, paritychecker, and
embedded probe lanes use an allowed EMEL-owned setup-time source contract
instead of actor-internal `io/read/detail.hpp`.

<rule_constraints>
- Benchmark and parity harnesses must not include actor `actions.hpp`,
  `guards.hpp`, `detail.hpp`, or `detail.cpp` helpers directly.
- `io/read` remains the actor-owned copy boundary and must not perform
  dispatch-time filesystem work.
- Maintained evidence may report `read_copy` only when the public model-loader,
  tensor, io-loader, and io-read path executes.
- Do not use a benchmark-regression override for validation.
</rule_constraints>

## Tasks

1. Add the public source contract.
   - Create `src/emel/io/source/any.hpp`.
   - Move setup-time file-byte loading semantics out of `io/read/detail.hpp`.

2. Rewire maintained lanes.
   - Update generation benchmark, Sortformer fixture, embedded probe, and
     paritychecker to include `emel/io/source/any.hpp`.
   - Replace `emel::io::read::detail::load_source_file_bytes` with
     `emel::io::source::load_file_bytes`.

3. Tighten guardrails.
   - Update model-loader source guardrails to require the public source API and
     forbid `emel/io/read/detail.hpp`.
   - Update paritychecker source guardrails to catch actor-internal include
     reach-through.

4. Validate.
   - Build focused test binaries.
   - Run model-loader, paritychecker, generation compare, domain boundary, and
     changed-file scoped quality gates.
