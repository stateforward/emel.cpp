## Phase Details

### Phase 81: Sortformer GGUF Fixture And Model Contract

**Goal:** Pin the exact maintained Sortformer GGUF artifact and make loader/model acceptance
truthful before any runtime work starts.

**Requirements:** SORT-01, SORT-02, SORT-03

**Success criteria:**
1. The maintained artifact records source URL, local path, checksum/provenance, expected
   architecture metadata, and fixture setup instructions.
2. The model loader accepts the maintained Sortformer contract and rejects incompatible,
   incomplete, or off-scope GGUF files with deterministic errors.
3. Required tensor families and streaming parameters are named in the model contract before
   execution can proceed.

### Phase 82: Diarization Request And Audio Frontend Contract

**Goal:** Establish one deterministic diarization input contract and native audio feature extractor for
the maintained model profile.

**Requirements:** DIA-01, DIA-02, DIA-03

**Success criteria:**
1. Valid requests accept single-channel `float32` PCM at 16,000 Hz without file decode, resampling,
   or channel mixing.
2. EMEL derives the maintained Sortformer acoustic feature/input tensor contract natively after
   one-time initialization.
3. Invalid sample rate, channel count, payload shape, chunk configuration, and output capacity fail
   through explicit diarization errors.

### Phase 83: Maintained Sortformer Contract Repair

**Goal:** Repair the maintained Sortformer artifact and request contracts so native execution work
starts from the real GGUF metadata and tensor families.

**Requirements:** RUN-02, RUN-03 prerequisites

**Success criteria:**
1. Maintained profile constants match the self-converted and maintained GGUF artifact.
2. The model contract names `prep.`, `enc.`, `mods.`, and `te.` tensor families.
3. The follow-on execution phases are stage-owned and explicitly reject synthetic/head-only
   substitutes.

### Phase 83.1: Sortformer Encoder Tensor Binding And Kernels

**Goal:** Bind and execute the maintained Conformer encoder slice natively without a generic
runtime dispatcher.

**Requirements:** RUN-01, RUN-03

**Success criteria:**
1. `src/emel/diarization/sortformer/encoder/` owns fixed-profile tensor views for the maintained
   `enc.l{N}.*` families and rejects incomplete encoder contracts.
2. Encoder numeric work is implemented in component-owned or kernel-owned helpers, not in tools,
   benchmarks, or request actions.
3. Focused tests prove encoder readiness, missing-tensor rejection, deterministic output, and
   allocation-free dispatch after construction.

### Phase 83.2: Sortformer Modules And Speaker Cache

**Goal:** Implement the maintained `mods.*` projection/head path and speaker cache state required
by streaming Sortformer execution.

**Requirements:** RUN-01, RUN-02, RUN-03

**Success criteria:**
1. `src/emel/diarization/sortformer/modules/` binds `mods.ep.*`, `mods.fh2h.*`, `mods.sh2s.*`,
   and `mods.h2s.*` families for the maintained profile.
2. `src/emel/diarization/sortformer/cache/` owns preallocated speaker-cache state with explicit
   cache-ready/cache-invalid guards.
3. Module/cache tests cover deterministic state updates, missing-family rejection, and no external
   fallback path.

### Phase 83.3: Sortformer Transformer Encoder Path

**Goal:** Execute the maintained `te.*` transformer encoder stack through fixed-profile native
stage code.

**Requirements:** RUN-01, RUN-02, RUN-03

**Success criteria:**
1. `src/emel/diarization/sortformer/transformer/` binds all maintained transformer layer tensor
   families and rejects incomplete layer contracts.
2. Transformer attention/feed-forward/norm numeric work is component-owned or kernel-owned and
   contains no runtime family dispatcher.
3. Focused tests prove deterministic transformer output and explicit SML error routing for invalid
   profile or missing cache inputs.

### Phase 83.4: Sortformer Executor Actor

**Goal:** Compose the maintained module/cache and transformer stages into one diarization-owned
native executor actor from validated encoder-frame input.

**Requirements:** RUN-01, RUN-02, RUN-03

**Success criteria:**
1. EMEL runs maintained `mods.*`, fixed speaker-cache writes, and maintained `te.*` transformer
   layers from EMEL-owned model and cache state.
2. Runtime choices for model readiness, tensor readiness, input shape, output capacity, and error
   outcomes are modeled in Boost.SML guards/states/transitions.
3. The executor exposes a deterministic native `188 x 192` hidden-state handoff for Phase 84
   without calling llama.cpp, ggml, ONNX Runtime, Python, NeMo, or tool-only helpers.

### Phase 83.5: Sortformer Feature-To-Encoder Execution Path

**Goal:** Complete the missing native Conformer encoder execution between prepared acoustic
features and the executor's validated `188 x 512` encoder-frame input.

**Requirements:** RUN-01, RUN-03

**Success criteria:**
1. EMEL derives `188 x 512` encoder frames from the maintained feature tensor using EMEL-owned
   encoder/pre-encoder tensor contracts instead of a synthetic projection.
2. Encoder-stage numeric work remains under `src/emel/diarization/sortformer/encoder/` or
   kernel-owned files, with no benchmark/tool fallback.
3. The executor can be widened to consume the native encoder-frame output without changing
   modules/cache/transformer behavior.

### Phase 84: Speaker Activity And Segment Output Contract

**Goal:** Publish deterministic diarization outputs that operators can compare, snapshot, and
consume.

**Requirements:** OUT-01, OUT-02, OUT-03

**Success criteria:**
1. EMEL emits a deterministic `T x 4` speaker-activity probability matrix with 0.08-second frame

2. EMEL emits bounded `speaker_0` through `speaker_3` segment records with stable labels,
   monotonic timestamps, and explicit overlap behavior.
3. Repeated runs on the same fixture/audio/profile produce byte-stable JSONL or snapshot-friendly
   records.

**Requirements:** BEN-01, RUN-03

### Phase 87: ARM Kernel Optimization Loop

**Goal:** Iterate on the measured ARM hotspots until additional optimizations are no longer

**Requirements:** RUN-03, BEN-01

**Success criteria:**
