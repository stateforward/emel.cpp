# v1.20 Milestone Audit

## Verdict

Status: gaps_found

The milestone cannot be closed. `.planning/ROADMAP.md` and `.planning/STATE.md` describe six
completed v1.20 phases, and `.planning/REQUIREMENTS.md` marks all 12 requirements complete, but
GSD phase discovery returns zero active phase directories and there is no
`.planning/milestones/v1.20-phases` archive. The workflow requires SUMMARY, VERIFICATION, and
VALIDATION evidence per phase; none exists for phases 167-172.

Source inspection found partial live support for the migration itself:

- `cmake/sml_version.cmake` pins `stateforward/sml.cpp` at
  `4a7109b5dd4aae40e78304e3ac03440ccc35031e`.
- `CMakeLists.txt` and `tools/docsgen/CMakeLists.txt` consume that pin through FetchContent.
- Active sampled source, tests, docsgen, and rules use `<stateforward/sml.hpp>` and
  `stateforward::sml`.
- A direct legacy include/namespace scan over `src include tests tools docs scripts
  .codex/get-shit-done cmake CMakeLists.txt` found no `boost/sml`, `<sml.hpp>`, or `boost::sml`
  matches.
- The integration checker reported standalone `emel_tests_sm`, `lint_snapshot`, and `generate_docs`
  CTest verification passed in `build/v120-sml`.

Those facts are useful but not enough for closeout. Artifact agreement is missing, and live
quality-gate wiring still contradicts the guardrail claim: `scripts/quality_gates.sh` leaves
`lint_snapshot` commented out, and the integration checker observed a scoped quality-gate benchmark
lane failure.

## Source-Backed Findings

### DEP-01 / DEP-02

Live source supports the dependency pin claim: `cmake/sml_version.cmake` sets
`EMEL_SML_GIT_REPOSITORY` to `https://github.com/stateforward/sml.cpp` and `EMEL_SML_GIT_TAG` to
`4a7109b5dd4aae40e78304e3ac03440ccc35031e`. Both root CMake and docsgen CMake include that file.
The requirement remains unsatisfied for milestone closeout because Phase 167 verification artifacts
are absent.

### SRC-01 / SRC-02 / DOC-01 / DOC-02 / DOC-03

Live source partially supports the migration: active sampled code and docs use
`stateforward::sml` and `<stateforward/sml.hpp>`, and a direct legacy SML scan found no `boost::sml`
or legacy include matches in the audited active paths. The requirement remains unsatisfied for
milestone closeout because the phase verification and summary cross-checks are missing.

### SRC-03

Live tests include state inspection and policy/introspection surfaces using `stateforward::sml`, for
example `tests/sm/sm_policy_tests.cpp`, `tests/sm/sm_any_tests.cpp`, and graph/text lifecycle tests.
The integration checker also found no active `make_dispatch_table` consumer and no active
`sml::logger` wiring/test under `src`, `tests`, or `tools`; logger and dispatch-table evidence is
docs-only. The requirement remains unsatisfied because Phase 169 verification artifacts,
quality-gate evidence, and complete live coverage are missing.

### DOC-01 / DOC-02 / DOC-03

The migration guidance is mostly updated to `stateforward::sml`, but active contributor guidance is
not fully consistent. `AGENTS.md` still says it is aligned with `docs/sml.rules.md`, and
`docs/plans/rearchitecture.plan.md` references `docs/sml.rules.md`; the normative file is
`docs/rules/sml.rules.md`. These stale paths keep the documentation requirements partial until
repaired and verified.

### VAL-01 / VAL-02 / VAL-03

These are source-backed blockers, not just artifact blockers:

- `scripts/quality_gates.sh` has a `lint_snapshot` step present but commented out.
- `scripts/check_sml_behavior_selection.sh` checks hidden behavior selection, but no dedicated
  source check was found that fails new unapproved legacy SML include or namespace references.
- The integration checker reported a scoped quality-gate run failed in the benchmark lane because
  no benchmark entries matched the selected suite and the benchmark snapshot gate failed.
- Closeout cannot be source-backed while this audit reports blockers.
