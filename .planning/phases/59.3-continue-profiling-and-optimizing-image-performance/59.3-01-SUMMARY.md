# Phase 59.3 Summary

## Outcome

Phase 59.3 is complete. The maintained single-thread image lane is materially faster than the
phase-entry baseline, and the current steady-state image hotspot profile is now cleanly narrowed:
kernel-owned pointwise work remains dominant, with depthwise second. The work stayed on the
public `embeddings/generator` image request path and did not add hidden control flow to the model.

## Delivered

- Kept the image path on one maintained steady-state benchmark surface and continued ROI-ranked
  profiling from the real `embeddings/generator` request flow.
- Landed the direct/fused image pointwise path and its follow-on data-plane tuning:
  - precomputed batch-norm affine state
  - SIMD batch-norm application
  - fused pointwise batch norm in the final store path
  - fused image stem/stage-0 standard-conv batch norm
  - reduced wasted output loads in the pointwise kernel
  - unrolled hot pointwise depth handling on the dominant image path
- Preserved focused parity coverage for the maintained pointwise, standard-conv, and depthwise
  image helpers.

## Measured Maintained Image Result

All numbers below exclude `initialize(...)` and come from the maintained image-only benchmark
surface with `EMEL_BENCH_CASE_FILTER=image`, `EMEL_BENCH_ITERS=10`, and `EMEL_BENCH_RUNS=5`.

- Original phase-entry image result from the captured context:
  - `237.262 ms/op`
  - `~4.21 embeddings/s`
- Current maintained q8 result:
  - `95.269 ms/op`
  - `~10.50 embeddings/s`
  - `prepare=4.650 ms`
  - `encode=90.616 ms`
  - `publish=0.001 ms`
- Current maintained q5 edge result:
  - `95.356 ms/op`
  - `~10.49 embeddings/s`
  - `prepare=4.738 ms`
  - `encode=90.470 ms`
  - `publish=0.001 ms`

Measured against the phase-entry baseline, the maintained image lane is now about `2.49x` faster.
