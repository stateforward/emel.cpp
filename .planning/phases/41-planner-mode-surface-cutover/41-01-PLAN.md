# Phase 41 Plan 01

## Objective

Canonicalize the `simple`, `sequential`, and `equal` planner-mode file surface so each mode has a
planner-owned canonical family under `src/emel/batch/planner/modes/`, while removing the leftover
shared `modes/detail.hpp` surface without changing maintained planner behavior.

## Tasks

1. Move the shared planner-mode helper implementation to a canonical planner-owned `detail.hpp`
   surface and retarget planner and mode headers to it.
2. Add canonical per-mode `context`, `detail`, `errors`, and `events` wrapper files so every mode
   directory exposes the allowed file bases without duplicating behavior logic.
3. Retarget tests and build files away from the deleted `modes/detail.hpp` surface and prove the
   planner-mode slice still builds and behaves the same.

## Verification

- `ctest --test-dir build/zig -R planner`
- `scripts/quality_gates.sh`

## Done Condition

Each planner mode directory exposes only canonical family files, the legacy shared mode-root
detail surface is gone, and focused planner verification still passes after the structural cutover.
