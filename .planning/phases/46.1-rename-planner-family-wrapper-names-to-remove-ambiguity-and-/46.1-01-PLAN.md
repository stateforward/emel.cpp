# Phase 46.1 Plan 01

## Objective

Rename the planner family's ambiguous wrapper-era names so the top-level planner and its direct
mode wrapper seams read as direct planner surfaces rather than alias layers.

## Tasks

1. Add a failing planner-surface regression that expects the renamed top-level planner event,
   state, and callable names and fails if the old wrapper-era names remain.
2. Rename the top-level planner event/runtime/state/callable surfaces and update the shared
   mode-wrapper helper plus direct maintained planner call sites.
3. Re-run focused planner tests and `scripts/quality_gates.sh`, then record the Phase 46.1
   summary, verification, roadmap, state, and audit updates.

## Verification

- `cmake --build build/zig --target emel_tests_bin -j4`
- `./build/zig/emel_tests_bin --no-breaks --source-file='*tests/batch/planner/*'`
- `scripts/quality_gates.sh`

## Done Condition

The planner family no longer exposes the ambiguous top-level wrapper-era names called out by the
user, the new regression proof locks the renamed surface in, and milestone bookkeeping reflects the
phase as complete with only the pre-existing benchmark warning tech debt remaining.
