# Plan 126.1: Whisper Recognizer Explicit Route Graph Repair

<rule_constraints>

## SML And Domain Rules

- No `sml::process_queue`, `sml::defer_queue`, mailbox, or self-dispatch.
- No runtime function-pointer route table, callback table, or runtime-indexed handler array for
  recognizer route behavior.
- Generic `src/emel/speech/recognizer/**` files must not expose Whisper/model-family names or
  contracts.
- Whisper-specific route support, readiness, encode, decode, and detokenize logic stays under
  `src/emel/speech/recognizer_routes/whisper/**`.
- Guard predicates decide support/readiness; actions execute already-selected route phases.
- Actions remain bounded and allocation-free during dispatch.

</rule_constraints>

## Goal

Make the maintained public recognizer path source-backed for `CLOSE-01` by replacing hidden
`runtime_backend` function-pointer dispatch with explicit, compile-time route graph integration
visible in `emel::speech::recognizer::sm`.

## Tasks

1. Replace the recognizer backend pointer contract.
   - Remove `event::runtime_backend` and `initialize.backend`.
   - Remove backend pointer storage from recognizer context.
   - Introduce a compile-time route policy surface used by recognizer guards and transition rows.

2. Move route predicates and route effects to explicit route surfaces.
   - Add Whisper route guard/effect headers under `speech/recognizer_routes/whisper`.
   - Keep support/readiness predicates in route guards.
   - Keep encode/decode/detokenize execution in route actions.

3. Update callers and tests.
   - Instantiate the maintained runner and Whisper fixture tests with the Whisper route policy.
   - Rewrite generic recognizer tests to use a test-only route policy without function pointers.
   - Add regression assertions that the recognizer path no longer exposes or invokes
     `runtime_backend`.

4. Verify and close the phase.
   - Run `scripts/check_sml_behavior_selection.sh` on the maintained recognizer path.
   - Run `scripts/check_domain_boundaries.sh`.
   - Run focused recognizer and Whisper recognizer tests.
   - Run maintained Whisper compare and benchmark suites.
   - Run changed-file scoped `scripts/quality_gates.sh`.
   - Write SUMMARY, VERIFICATION, and VALIDATION artifacts with source-backed evidence.

## Acceptance

- `scripts/check_sml_behavior_selection.sh src/emel/speech/recognizer src/emel/speech/recognizer_routes/whisper src/emel/speech/encoder/whisper src/emel/speech/decoder/whisper src/emel/speech/tokenizer/whisper` exits 0.
- Generic recognizer leak checks still find no Whisper/model-family names in
  `src/emel/speech/recognizer` or `tests/speech/recognizer`.
- Recognizer-backed compare remains exact-match `[C]`.
- Recognizer-backed benchmark remains `ok` with EMEL faster than the matched reference lane.
