# Phase 48 Plan 01

## Objective

Accept truthful `omniembed` GGUF metadata and bind a narrow internal multimodal execution contract
without routing the model through the decoder-only LLM path.

## Tasks

1. Register `omniembed` in the model architecture registry and add a dedicated
   `src/emel/model/omniembed` family with hparam loading for shared embedding length, modality
   encoder dimensions, and Matryoshka truncation metadata.
2. Implement a dedicated `omniembed` execution-contract validator that explicitly requires the
   text/image/audio encoder and projection tensor families and fails malformed layouts
   deterministically.
3. Add loader/model contract tests for successful `omniembed` GGUF acceptance, internal contract
   binding, and failure when a required modality family is missing; then run focused model tests and
   full quality gates before closing the phase.

## Verification

- `cmake --build build/zig --target emel_tests_bin -j4`
- `./build/zig/emel_tests_bin --no-breaks --source-file='*tests/model/*'`
- `scripts/quality_gates.sh`

## Done Condition

`emel` recognizes `gguf.architecture=omniembed`, loads the maintained shared embedding metadata,
and validates a multimodal tensor-family contract explicitly through its own architecture family
instead of silently reusing a decoder-only execution path.
