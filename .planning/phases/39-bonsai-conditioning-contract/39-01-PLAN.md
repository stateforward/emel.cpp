# Phase 39 Plan 01

## Objective

Make the maintained Bonsai request path derive one explicit structured formatter contract from the
loaded model metadata and reject every unsupported request shape before formatting/tokenization.

## Tasks

1. Move the supported embedded-template contract markers and formatter logic into `src` so the
   runtime owns the supported Qwen/Bonsai chat contract.
2. Propagate an explicit formatter contract kind through generator and conditioner binding, and use
   conditioner guards to reject unsupported message roles, disabled generation prompts, thinking,
   and unsupported template variants.
3. Populate `model_data.meta.tokenizer_data` from GGUF metadata in the maintained bench/parity
   loaders so runtime contract resolution follows the embedded template instead of stale tool-only
   inference.
4. Add focused doctests that lock formatter resolution, supported Qwen formatting, and conditioner
   rejection behavior for unsupported request shapes.
