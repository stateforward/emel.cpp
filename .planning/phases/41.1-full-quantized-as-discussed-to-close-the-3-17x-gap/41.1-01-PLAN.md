# Phase 41.1 Plan 01

## Objective

Remove the biggest remaining non-native Bonsai runtime seams in `src/emel` so the maintained
`Q1_0_g128` slice is materially faster before publication and no longer reports quantized token
embedding as an approved dense fallback.

## Tasks

1. Port the Prism/ggml ARM `Q1_0_g128 x q8_0` vector-dot behavior into EMEL-owned AArch64 kernel
   guards/actions/detail so vector matmul and argmax stop using the current scalar fallback on the
   shipped runtime lane.
2. Tighten the shipped token-embedding path so quantized embeddings are treated as a native runtime
   stage, not `approved_dense_f32_by_contract`, while keeping control flow explicit and bounded.
3. Add narrow regression coverage for the new `Q1_0_g128` optimized route plus the updated
   quantized-stage contract, then re-measure the isolated Bonsai implementation bench and keep the
   full quality gate green.

## Verification

- `cmake --build build/zig --parallel --target emel_tests_bin`
- `./build/zig/emel_tests_bin --test-case='*q1_0_g128*' --no-breaks`
- `./build/zig/emel_tests_bin --test-case='*quantized_path_audit*' --no-breaks`
- `./scripts/paritychecker.sh --implementation --model tests/models/Bonsai-1.7B.gguf --text hello --max-tokens 1`
- `./scripts/bench.sh --implementation --model tests/models/Bonsai-1.7B.gguf --system`
- `./scripts/quality_gates.sh`

## Done Condition

The maintained Bonsai runtime no longer reports quantized token embedding as an approved dense
fallback, the shipped AArch64 `Q1_0_g128` hot path is materially more native than the previous
scalar implementation, the isolated Bonsai implementation bench improves from the prior `3.173x`
ratio, and parity plus `quality_gates` stay green.
