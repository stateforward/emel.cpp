# Plan 154-01: Generation Live Reference Truth Closure

<rule_constraints>
- Keep EMEL and reference runtime state separate; do not share llama-created model/vocab/tokenizer
  objects with the EMEL lane.
- Do not use stored baselines as the primary reference truth.
- Keep snapshot baseline handling append-only and source-backed by the live reference result.
- Preserve existing missing-model and unsupported-fixture failure semantics.
</rule_constraints>

## Goal

Make maintained generation parity prove EMEL output against the live reference runtime result on the
normal success path while keeping stored baselines as publication artifacts.

## Tasks

1. Add a failing guard/test for the current baseline-as-reference path.
2. Wire `run_generation_harness_contract(...)` to run `run_reference_generate(...)` and compare EMEL
   against that live result.
3. Move baseline handling after live comparison and compare loaded baselines against live reference
   output for stale-artifact detection.
4. Update generation output markers/tests so live reference implementation and baseline artifact
   metadata remain distinguishable.
5. Run focused paritychecker tests and changed-file scoped quality gates.
