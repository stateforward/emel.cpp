# Phase 59.4 Plan 01

## Objective

Move the maintained image hot path into AArch64 kernel code, rewire the image runtime to that
kernel layer, and publish updated single-thread image-throughput evidence.

## Tasks

1. Add the phase artifacts plus focused kernel-side request/verification scaffolding for the image
   pointwise hot path, reusing the existing bind-time prepack contract.
2. Move the fused pointwise `1x1` ARM hot loop from `src/emel/embeddings/generator/detail.hpp`
   into `src/emel/kernel/aarch64/actions.hpp`, then rewire the maintained image runtime to call
   that kernel entry point without reintroducing hidden control flow.
3. If the implementation stays bounded and the fresh profile still justifies it, move the hot
   image depthwise `3x3` path into the same kernel layer; then rerun the maintained image bench,
   focused tests, and full gate path before closing out the phase.
