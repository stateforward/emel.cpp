# Phase 59.1.1 Plan 01

## Objective

Run the agreed `llama.cpp`-supported ARM baseline matrix on this same host and publish the gap
between EMEL and the external reference lanes by modality and available stage timing.

## Tasks

1. Extend the maintained reference benchmark surface so it can measure the approved matrix:
   add text embedding cases for `Arctic S` and `EmbeddingGemma 300M`, keep the existing multimodal
   image/audio lane for `LFM2-VL-450M` and `Ultravox 1B`, and print per-case metadata that makes
   the producing model explicit.
2. Add or update the maintained reference-run entrypoint and asset contract so the full matrix can
   be built and run reproducibly on the same ARM host, including exact environment variables and
   any newly pinned text-model provenance.
3. Run the reference matrix that is locally supportable on this host, record the resulting
   embeddings/s and stage timings beside the maintained EMEL numbers, and publish an explicit gap
   table plus any still-missing asset blockers for the remaining cases.

## Verification

- `scripts/bench_embedding_reference_liquid.sh --build-only`
- `cmake --build build/bench_tools_liquid_ninja --target embedding_reference_bench_runner -j4`
- `EMEL_BENCH_CASE_FILTER=text ./build/bench_tools_liquid_ninja/embedding_reference_bench_runner`
- `EMEL_BENCH_CASE_FILTER=image ./build/bench_tools_liquid_ninja/embedding_reference_bench_runner`
- `EMEL_BENCH_CASE_FILTER=audio ./build/bench_tools_liquid_ninja/embedding_reference_bench_runner`
- `EMEL_BENCH_ITERS=10 EMEL_BENCH_RUNS=5 ./build/bench_tools_liquid_ninja/embedding_reference_bench_runner`
- `scripts/quality_gates.sh`

## Done Condition

The phase closes only when the same-host reference benchmark publishes real ARM throughput evidence
for every locally supportable model in the approved matrix, the gap to EMEL is written down
truthfully by modality and stage, and any remaining missing cases are blocked by explicit external
asset absence rather than missing in-repo benchmark support.
