# Phase 59.1.1 Summary

Phase `59.1.1` is complete. The maintained reference benchmark lane now runs the agreed same-host
ARM comparison matrix for `Arctic S`, `EmbeddingGemma 300M`, `LFM2-VL-450M`, and `Ultravox 1B`,
with explicit text/image/audio cases, reproducible asset bootstrapping, and per-case

## Measured Same-Host ARM Reference Results

All numbers below exclude initialization and model load, and come from isolated steady-state runs
with `EMEL_BENCH_ITERS=10` and `EMEL_BENCH_RUNS=5`.

- `Arctic S` text reference:
  - `~82.0 embeddings/s` (`12.195 ms`)
  - `prepare=0.0056 ms`, `encode=12.187 ms`, `publish=0.00015 ms`
  - output anchor: `1 x 384`
- `EmbeddingGemma 300M` text reference:
  - `~7.58 embeddings/s` (`131.957 ms`)
  - `prepare=0.0626 ms`, `encode=131.890 ms`, `publish=0.00026 ms`
  - output anchor: `1 x 768`
- `LFM2-VL-450M` vision reference:
  - `~2.78 requests/s` (`359.297 ms`)
  - `prepare=0.449 ms`, `encode=358.542 ms`, `publish=0.0042 ms`
  - output anchor: `64 x 1024`
- `Ultravox 1B` audio reference:
  - `~0.164 requests/s` (`6102.235 ms`)
  - `prepare=10.701 ms`, `encode=6088.753 ms`, `publish=0.0304 ms`
  - output anchor: `187 x 2048`

## Gap To Maintained EMEL Results

Compared against the Phase `59.1` EMEL steady-state numbers on the same host:

- Text:
  - EMEL: `~169.5 embeddings/s` (`5.901 ms`)
  - EMEL is `~2.07x` faster than the `Arctic S` reference lane.
  - EMEL is `~22.4x` faster than the `EmbeddingGemma 300M` reference lane.
- Image:
  - EMEL: `~1.51 embeddings/s` (`661.395 ms`)
  - EMEL is still behind the `LFM2-VL-450M` tower lane by `~1.84x` on per-request latency.
- Audio:
  - EMEL: `~44.9 embeddings/s` (`22.269 ms`)
  - EMEL is `~274x` faster than the `Ultravox 1B` reference lane on this same-host request.

## Comparison Caveats

- The text references are the cleanest comparison: both publish one final sequence embedding per
  request.
- The multimodal references are only the closest truthful external comparison, not exact output
  parity with EMEL. `LFM2-VL-450M` and `Ultravox 1B` publish media-token embedding grids from the
  `mtmd` lane (`64 x 1024` for image, `187 x 2048` for audio), while the maintained EMEL lane
  publishes one final normalized embedding vector per request.
- Because the model families, parameter counts, and output contracts differ materially, the
  multimodal numbers should be read as directional tower-throughput baselines rather than strict
  embedding-head parity claims.

## Remaining Signal

- Image is still the weakest EMEL modality relative to the chosen external reference; the next ARM
  throughput pass should stay focused on the vision encode tower.
- Audio no longer needs an urgent same-host throughput rescue relative to the chosen external
  baseline, though it still has room to improve in absolute terms if the product target demands it.
