# Plan 145-01: Native Quantized Route Evidence Closure

## Goal

Close the remaining v1.17 audit gaps for text generator native quantized route ownership and
evidence.

## Tasks

1. Add a regression that scans the actual `matmul_vector_native_quantized(...)` helper for hidden
   packed-q8/q8-k route probing.
2. Remove dispatch-time native quantized route probing from action-called detail code while keeping
   prepared tensor layouts as the source of optimized kernel dispatch.
3. Replace `phase_lifecycle(...)` runtime-indexed manifest selection with explicit prefill/decode
   lifecycle binding helpers called from compile-time selected action paths.
4. Run focused generator and parity tests, domain boundary checks, and the changed-file scoped
   quality gate.
5. Record validation evidence and update roadmap state truthfully.

## Success Criteria

- No action-called detail helper selects packed-q8, q8-k, native quantized, or kernel matmul
  fallback paths at dispatch time.
- Existing text generator route guards remain the source of externally visible runtime route
  selection.
- The regression catches the previously missed native quantized helper body.
- Native quantized diagnostics and generation tests keep passing.
- Phase evidence includes exact commands and results.
