# Phase 59.1 Plan 01

## Objective

Raise steady-state ARM embeddings throughput in the maintained generator runtime using ROI-ranked
profiling, anchored to the current `prepare` / `encode` / `publish` split and the agreed
`llama.cpp`-supported comparison matrix.

## Tasks

1. Optimize the image `encode` path first by using the maintained stage split to attack the
   remaining scalar spatial/depthwise conv and layout churn in the vision tower without violating
   the no-fallback kernel contract.
2. Optimize the audio `prepare` path second by precomputing and reusing invariant frontend state
   and vectorizing the hottest resample/STFT/mel/log loops without changing the maintained audio
   request contract.
3. Reprofile text/image/audio on the maintained benchmark, confirm whether shared kernel/layout
   work moved text enough to warrant a dedicated pass, and publish the final ARM evidence against
   the agreed comparison anchors: `Arctic S`, `EmbeddingGemma 300M`, `LFM2-VL-450M`, and
   `Ultravox 1B`.

## Done Condition

Image `encode_ns` and audio `prepare_ns` both improve measurably on the maintained ARM benchmark,
any text gains are either captured or explicitly deferred with evidence, and the phase closeout
records the current gap against the agreed external baseline set.
