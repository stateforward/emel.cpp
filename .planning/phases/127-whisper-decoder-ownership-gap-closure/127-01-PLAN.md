# Plan 127.1: Whisper Decoder Ownership Gap Closure

<rule_constraints>

- Keep Boost.SML orchestration unchanged except where explicit transition visibility is required.
- Keep runtime choice in guards and `sm.hpp`; actions may only run an already-selected decoder
  variant.
- Keep decoder runtime contracts speech-owned; do not expose `emel::model::whisper` from decoder
  public events, context, guards, or actions.
- Keep model-family-specific code under existing speech/model variant boundaries only.

</rule_constraints>

## Tasks

1. Move decoder runtime helpers into decoder ownership.
   - Extend `src/emel/speech/decoder/whisper/detail.hpp` with decoder-owned tensor lookup,
     quantized read, linear, cross-attention cache, logits, timestamp-aware token selection, and
     decode-sequence helpers needed by decoder actions and guards.
   - Do not change public decoder event contracts.

2. Remove encoder-detail dependencies from decoder code.
   - Update `src/emel/speech/decoder/whisper/actions.hpp` to use decoder detail types and
     `detail::run_decoder_sequence`.
   - Update `src/emel/speech/decoder/whisper/guards.hpp` to use decoder detail tensor helpers.
   - Verify `rg -n "encoder/whisper/detail|encoder::whisper::detail" src/emel/speech/decoder/whisper`
     returns no matches.

3. Preserve maintained E2E behavior.
   - Build and run focused decoder, recognizer, compare, and benchmark evidence.
   - Keep compare exact-match transcript `[C]` and benchmark EMEL faster than reference.

4. Close planning evidence.
   - Record SUMMARY, VERIFICATION, and VALIDATION artifacts for Phase 127.
   - Update `ROADMAP.md`, `REQUIREMENTS.md`, and `STATE.md` only after source-backed checks pass.
