# Phase 236 Plan 01: Publication and Evidence Truthfulness

## Objective

Satisfy `DOC-01`, `LNT-01`, `BNH-01`, and `EVI-01` by ensuring checked-in docs and
artifacts are strictly aligned with maintained source behavior and recorded validation.

## Steps

1. Trace staged-read claims from maintained docs/state to source entrypoints.
   - Verify how staged-read is selected/reported in loader, bench, paritychecker, and probe lanes.
   - Confirm wording does not overstate staged runtime execution.

2. Audit lint evidence truth (`LNT-01`).
   - Determine whether current phase edits require lint snapshot update.
   - If no update required, prove this from changed files/status and keep snapshots unchanged.
   - If required, use maintained lint workflow only and record exact command/result.

3. Audit benchmark snapshot truth (`BNH-01`).
   - Determine whether staged-read-affected benchmark snapshot updates are required from current source/status.
   - If required, use maintained benchmark snapshot workflow and record exact commands/results.
   - If not required, record explicit source/status justification.

4. Audit evidence labels (`EVI-01`).
   - Verify parity/benchmark outputs and related metadata do not label runs as staged-read-backed
     unless staged runtime path actually executes.

5. Update phase and milestone docs with only source-backed statements.
   - Produce `236-01-SUMMARY.md` and `236-VERIFICATION.md`.
   - Update `ROADMAP.md`, `REQUIREMENTS.md`, and `STATE.md` only where truth correction is needed.

## Validation Plan

- Source checks (`rg`) over maintained surfaces to prove claim-to-source linkage.
- Focused test/lint/benchmark commands only when required by truthful closeout:
  - Lint: maintained `lint_snapshot` path if needed.
  - Bench snapshot workflow: maintained update path if needed.
- Record every command and exit code in `236-VERIFICATION.md`.
- Explicitly document residual risk and any not-run lanes as not claimed.

## Out of Scope

- Runtime feature work beyond narrow truthfulness fixes.
- Snapshot churn without maintained-command necessity.
- Root-checkout artifacts as evidence.
