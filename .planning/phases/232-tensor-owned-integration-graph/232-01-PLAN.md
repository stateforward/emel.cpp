## Goal

Land tensor-owned staged-read integration graph with explicit public-event
dispatch and explicit done/error terminals.

## Requirements

- TNX-01
- TNX-02
- TNX-03
- TNX-04

## Tasks

1. Extend tensor public events:
   - add `request_staged_load`,
   - add `request_staged_load_done`,
   - add `request_staged_load_error`.
2. Extend tensor integration context/status/runtime:
   - inject `io::staged_read::sm*` in `action::context`,
   - add staged runtime/status in `detail.hpp`,
   - add staged error taxonomy in `errors.hpp`.
3. Add staged integration guards/actions/state graph:
   - validate staged request contract and actor availability in guards,
   - dispatch only via `io::staged_read::event::staged_window`,
   - map staged outcomes to explicit tensor terminal states/events.
4. Add `process_event(const event::request_staged_load&)` wrapper with
   stack-scoped runtime status.
5. Add focused doctests in `tests/model/tensor/lifecycle_tests.cpp`.
6. Run requested validation commands and document exits/evidence.
