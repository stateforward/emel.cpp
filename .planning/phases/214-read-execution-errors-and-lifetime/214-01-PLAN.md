# Phase 214 Plan 01: Read Execution, Errors, and Lifetime

## Goal

Replace the Phase 213 read-attempt placeholder with concrete file open, seek, read, close,
success publication, and deterministic execution errors while preserving tensor residency
ownership outside the read actor.

## Requirements Closed

| REQ-ID | Source-Backed Evidence Plan |
|--------|------------------------------|
| READ-03 | Successful read copies requested bytes into caller-owned target buffer and publishes `_done`; status data is same-RTC only. |
| LIFE-01 | Platform resource is closed inside the attempt before `_done`; no handle is retained in context. |
| ERR-01 | Explicit error categories/states exist for invalid, unsupported, open, seek, read, short read, and internal errors. |

## Tasks
