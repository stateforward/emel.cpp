# Phase 42 Plan 01

## Objective

Replace the hidden planner-to-mode reach-through with explicit mode request and outcome events plus
mode wrapper dispatch, while preserving the external planner request and callback contract.

## Tasks

1. Define mode-local request and `_done` / `_error` outcome events for `simple`, `equal`, and
   `sequential`.
2. Add wrapper `process_event(...)` surfaces on each mode machine that accept the mode-local
   request and translate the internal planner runtime into typed mode outcome events.
3. Retarget the top-level planner to dispatch into mode wrappers via explicit typed events and
   branch on explicit mode outcome completion instead of direct child `request_runtime`
   completion.
4. Extend planner-family tests to lock the new event boundary surface.

## Verification

- `./build/zig/emel_tests_bin --no-breaks --source-file='*tests/batch/planner/*'`
- `scripts/quality_gates.sh`

## Done Condition

Planner-to-mode handoff is readable through explicit typed wrapper events, mode outcomes use
contract-aligned `_done` / `_error` names, and focused planner verification still passes.
