# Plan 245-01: Block Geometry Ownership and Slot Allocation

**Phase:** 245
**Requirements:** KVM-01, KVM-02
**Status:** In progress (2026-07-04)

## Tasks

1. **Shared geometry contract in `emel::memory::view`** (KVM-01)
   - `DEFAULT_BLOCK_TOKENS`, `MAX_BLOCKS`, `blocks_for_tokens`,
     `positions_capacity_for`, `resolved_or_default` in `src/emel/memory/view.hpp`.
   - `memory/kv/detail.hpp` constants alias the view contract; kv actions/guards
     `blocks_for_length` delegates to the shared helper (removes two duplicates).

2. **Ascending block-id allocation** (KVM-01 prerequisite for Phase 248)
   - `reset_runtime` fills the free stack reversed so fresh allocations pop
     ascending contiguous ids; frees still push LIFO.
   - `memory_kv_lifecycle_mapping_order_is_deterministic` updated to the
     ascending contract plus multi-block contiguity assertions.

3. **Backend consumes geometry at prepare** (KVM-01)
   - `native_backend.kv_block_tokens` / `kv_positions_capacity`; `prepare` takes
     `kv_block_tokens` (default = contract default) and derives block-padded
     per-layer position capacity.
   - Cache sizing loop, linear/flash offset helpers, and flash kernel view head
     strides switch extent from `n_ctx` to `kv_positions_capacity`
     (bit-identical when `n_ctx % block_tokens == 0`).

4. **Generator geometry coverage guard** (KVM-01, KVM-02)
   - The public contract keeps requiring explicit positive geometry
     (`valid_initialize`); `limits` carries it verbatim and
     `request_backend_prepare` forwards `limits.block_tokens` into `prepare`.
   - New pure guards `guard_memory_geometry_covers_context` /
     `guard_memory_geometry_gap` on the `reserving_memory` decision route
     geometry gaps to `mark_invalid_request` before any reserve dispatch.

5. **Tests** (KVM-02 proof)
   - kv machine: ascending mapping + existing `block_oom` exhaustion coverage.
   - Generator machine: geometry-gap initialize rejection; zero-geometry
     requests rejected by the public contract.
   - detail: prepare derives default/divisible/padded geometry; padded extents.
   - Hand-built test backends now set the geometry fields they previously
     implied (13 sites, tests only).
