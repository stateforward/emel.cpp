# Phase 59.1.1.1 Plan 01

## Objective

Raise maintained single-thread ARM image throughput by replacing the current per-pixel pointwise
`1x1` lowering with a batched matrix path, then rebenchmark and reprofile before taking any
second optimization step.

## Tasks

1. Add the phase artifacts and the minimum one-time image workspace needed to support a batched
   pointwise lowering without dispatch-time allocation or orchestration changes.
2. Replace the per-pixel `pointwise_conv_hwc(...)` path with a batched/tiled lowering that routes
   the image tower through the maintained f32 matmul kernel in larger matrix blocks, then wire the
   image callers to that path on the maintained runtime.
3. Rebuild, rerun the maintained image benchmark, and reprofile the single-thread image case to
   decide whether a second in-phase optimization is justified; if not, close out with the new
   measurements and hotspot shift evidence.

## Done Condition

The maintained image benchmark improves measurably, the hotspot stack shifts away from the
per-pixel pointwise lowering, and the phase closeout records whether the next ROI target is still
inside this phase or intentionally deferred.
