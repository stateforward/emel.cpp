# Phase 59.1.1.1 Summary

## Outcome

Phase 59.1.1.1 is complete. The maintained single-thread image-tower path now batches the
highest-ROI pointwise `1x1` work through the existing f32 matmul kernel instead of issuing one
tiny generic matmul per pixel. The phase stayed on the maintained
`src/emel/embeddings/generator` request path, improved steady-state image throughput materially,
and shifted the next image ROI away from the old pointwise lowering seam.

## Delivered

- Added one-time `image_pointwise_workspace` scratch storage so the new image lowering does not
  allocate during dispatch.
- Added dense matrix views and a maintained batched f32 matrix path over the existing kernel lane
  for tiled pointwise image work.
- Added tiled HWC pack, execute, and unpack helpers for pointwise `1x1` image conv lowering.
- Wired the tiled pointwise path into the maintained image tower for the edge-residual pointwise
  stage, universal inverted-block `pw_exp`, universal inverted-block `pw_proj`, and the stage-4
  pointwise conv.
- Left the final head pointwise path unchanged because its pixel count is `1`, so the batched
  lowering has no meaningful ROI there.

## Measured Steady-State ARM Result

All numbers below exclude `initialize(...)` and come from the maintained image-only benchmark
surface with `EMEL_BENCH_CASE_FILTER=image`, `EMEL_BENCH_ITERS=10`, and `EMEL_BENCH_RUNS=5`.

- Before this phase: `661.395 ms/op` (`~1.51 embeddings/s`)
- After this phase: `234.083 ms/op` (`~4.27 embeddings/s`)
- Stage split after the landed change:
  - `prepare=4.577 ms`
  - `encode=229.504 ms`
  - `publish=0.001 ms`

That is a `~2.83x` steady-state single-thread image-throughput improvement on the maintained ARM
path.

## Post-Change Hotspot Shift

The focused post-change sample shows the old per-pixel pointwise lowering is no longer the next
separate ROI target:

- `~88%` of the obvious hot samples now land in `execute_neon_mul_mat(...)`
- `~4%` land in `pointwise_conv_hwc_tiled_f32(...)`
- `~3%` land in `depthwise_conv_hwc(...)`
- `~1%` to `2%` land in `standard_conv_hwc(...)`
- edge-residual is no longer a meaningful next isolated hotspot

The next image-specific ROI is now depthwise work, not the old pointwise `1x1` lowering seam.
