# Phase 49 Plan 01

## Objective

Run the maintained TE text tower from conditioned text input through the `omniembed` contract and
return a normalized shared-space embedding with supported Matryoshka truncation.

## Tasks

1. Add `src/emel/embeddings/generator/` as the embedding-session actor for the maintained text
   lane, including initialization/bind flow for the existing text conditioner and a text embedding
   request surface that stays internal to `src/`.
2. Implement the TE text runtime on the real GGUF tensor families:
   `LEAF-IR BertModel -> mean pooling -> dense(384 -> 768) -> normalize -> TE projection head
   (768 -> 1920 -> 1280) -> normalize`, with optional truncation to the supported Matryoshka
   dimensions and renormalization.
3. Add maintained-fixture tests that load `tests/models/TE-75M-q8_0.gguf`, drive the new text
   lane on canonical text prompts, and verify normalized `1280`-dim output, supported truncation,
   and explicit invalid-dimension rejection; then run focused tests and full quality gates before
   closing the phase.

## Verification

- `cmake --build build/zig --target emel_tests_bin -j4`
- `./build/zig/emel_tests_bin --no-breaks --source-file='*tests/embeddings/*'`
- `./build/zig/emel_tests_bin --no-breaks --source-file='*tests/model/*'`
- `scripts/quality_gates.sh`

## Done Condition

EMEL can accept maintained text input on `TE-75M-q8_0.gguf`, run the real TE text embedding path,
and publish a normalized shared-space embedding with supported Matryoshka truncation, all through a
repo-owned embedding actor rather than a generation-only runtime surface.
