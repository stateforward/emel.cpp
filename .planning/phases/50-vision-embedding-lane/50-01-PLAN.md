# Phase 50 Plan 01

## Objective

Extend the maintained `embeddings/generator` actor with a truthful in-memory image lane that runs
the real TE vision tower (`MobileNetV4-Medium`) plus the shared projection head and returns a
normalized `1280`-dimensional embedding.

## Tasks

1. Make the `omniembed` model contract expose the maintained vision preprocessing truth by loading
   `omniembed.image_encoder_name` and deriving the official `mobilenetv4_conv_medium.e180_r384_in12k`
   input contract (`384 x 384`, bicubic resize, ImageNet mean/std) into repo-owned model metadata,
   with loader tests proving the derived fields.
2. Extend `src/emel/embeddings/generator/` with an explicit `embed_image` request path and
   image-specific guards/actions that validate one bounded RGBA memory contract, preprocess the
   image into the MobileNetV4 input layout, run the real `image_encoder.*` conv stack and
   `image_projection.*` head, and publish the normalized shared-space embedding through the
   existing result contract.
3. Add maintained image tests that synthesize the canonical `red-square` RGBA payload in memory and
   verify normalized `1280`-dimensional output plus explicit invalid-payload rejection, then run
   focused tests and full quality gates until the whole repo is green.

## Verification

- `cmake --build build/zig --target emel_tests_bin -j4`
- `./build/zig/emel_tests_bin --no-breaks --source-file='*tests/embeddings/*'`
- `./build/zig/emel_tests_bin --no-breaks --source-file='*tests/model/*'`
- `scripts/quality_gates.sh`

## Done Condition

EMEL accepts the maintained in-memory `red-square` RGBA payload, preprocesses it according to the
declared MobileNetV4 encoder contract, runs the TE vision tower and image projection natively in
`src/`, returns a normalized `1280`-dimensional embedding, and rejects malformed image payloads
explicitly while keeping the repo-wide gates green.
