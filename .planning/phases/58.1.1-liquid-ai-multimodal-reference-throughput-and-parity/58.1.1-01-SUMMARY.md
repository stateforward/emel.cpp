# Phase 58.1.1 Summary

## Outcome

Phase 58.1.1 is complete. The repo now has a maintained Liquid AI multimodal reference lane under
`tools/bench` that builds against the pinned `Liquid4All/benchmarks-llama.cpp` fork and exposes a
skip-safe C++ benchmark runner for image/audio tower timing and parity anchors.

## Delivered

- Added `tools/bench/embedding_reference_bench.cpp` and
  `tools/bench/embedding_reference_bench_main.cpp`.
- Added `scripts/bench_embedding_reference_liquid.sh` to configure a dedicated
  `build/bench_tools_liquid_ninja` build pinned to:
  - repository: `https://github.com/Liquid4All/benchmarks-llama.cpp.git`
  - ref: `0f345f3a1924ef274678a65bdec4b17f14e8fe9a`
- Extended `tools/bench/CMakeLists.txt` with a standalone `embedding_reference_bench_runner`
  target that links the fetched reference repo's `mtmd`, `llama`, and `ggml` targets without
  changing the existing EMEL embedding-generator runner.
- Extended `tools/bench/bench_common.hpp` so dedicated embedding-style runners can publish output
  anchors alongside stage timing.

## Reference Runner Contract

- Measures `prepare`, `encode`, and `publish` timing separately.
- Uses `libmtmd` directly rather than shelling out to `llama-mtmd-cli`.
- Emits deterministic output anchors:
  - `output_tokens`
  - `output_dim`
  - `output_checksum`
- Uses maintained synthetic inputs that match the EMEL embedding benchmark intent:
  - image: RGB `32x32` red square
  - audio: mono `f32` `16 kHz` `4000`-sample `440 Hz` pure tone
- Skips cleanly when `EMEL_REFERENCE_MM_MODEL` and `EMEL_REFERENCE_MM_MMPROJ` are not configured
  to existing compatible multimodal assets.

## Current Limitation

This phase establishes the maintained Liquid reference lane, but it does not yet provide reference
throughput numbers in this repo because the current environment does not have a compatible Liquid
multimodal text-model plus projector pair configured through:

- `EMEL_REFERENCE_MM_MODEL`
- `EMEL_REFERENCE_MM_MMPROJ`

The runner now makes that missing prerequisite explicit instead of forcing ad hoc comparison work.
