# Phase 58.1.1 Context

## Phase Boundary

Phase 58.1.1 adds the missing high-quality multimodal reference lane for the current ARM
throughput work. The current EMEL benchmark is honest, but the existing benchmark reference seam is
still centered on upstream llama.cpp text/generation surfaces. This phase adds a maintained
comparison path against Liquid AI's `benchmarks-llama.cpp`, which exposes `libmtmd`,
`llama-mtmd-cli`, and explicit image/audio tower support.

## Implementation Decisions

### Scope
- Keep the new comparison lane inside `tools/bench`.
- Build the Liquid reference surface through the existing reference-fetch seam, but isolate it from
  the current EMEL-only embedding-generator runner.
- Measure the reference image/audio tower path directly through `libmtmd`, not through shell-driven
  CLI parsing or Python wrappers.

### Reference Contract
- Use `Liquid4All/benchmarks-llama.cpp` as the maintained multimodal reference repository for this
  phase.
- Report reference `prepare`, `encode`, and `publish` timing separately, aligned with the EMEL
  stage split already added in Phase `58.1`.
- Include deterministic output anchors such as output token count, embedding dimension, and a
  checksum of the encoded output buffer so the runner is useful for both throughput and parity
  regression tracking.

### Asset And Compatibility Reality
- The local repo does not currently ship a Liquid/`mtmd`-compatible multimodal model + projector
  pair, so the maintained runner must handle missing reference assets gracefully.
- The maintained input assets can still reuse the local red-square image pattern and pure-tone
  waveform shape for reference-side media construction.
- The workflow should capture a maintained build path for the Liquid runner even if the current
  environment cannot execute a full reference measurement immediately.

## Existing Code Insights

### Current Reference Seam
- `tools/bench/CMakeLists.txt` already fetches one reference implementation via `REF_IMPL_REPOSITORY`
  and `REF_IMPL_REF`.
- The fetched reference source exposes `tools/mtmd/CMakeLists.txt`, `mtmd.h`, and `mtmd-helper.h`,
  so the benchmark can link directly against `mtmd`.

### Relevant Liquid-Fork Evidence
- The Liquid fork README lists `LFM2-VL` under supported multimodal models.
- Its multimodal docs state support for both image and audio input.
- Its source tree contains `tools/mtmd/mtmd-audio.cpp`, `tools/mtmd/mtmd-image.cpp`,
  `tools/mtmd/clip-model.h`, and `llama-mtmd-cli`.

## Specific Ideas

- Add a dedicated `embedding_reference_bench_runner` target that links `mtmd`, `llama`, and `ggml`
  from a Liquid-configured reference build.
- Create a maintained script that configures a dedicated build directory pinned to the Liquid fork
  and builds the reference runner without disturbing existing benchmark builds.
- Use environment variables for the reference model and projector paths so the runner can skip
  cleanly when compatible artifacts are absent, while still exposing the configured reference repo
  and commit.

## Deferred Ideas

- Automatic Hugging Face download/bootstrap for Liquid multimodal assets
- A separate `tools/paritychecker` C++ tool for multimodal output diffing
- Full EMEL vs Liquid side-by-side output comparison on the same compatible model family
