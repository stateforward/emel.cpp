# Phase 25.1: Restore Canonical Flash-Attention Dispatch On The Maintained Generator Path - Context

**Gathered:** 2026-03-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 25.1 must restore the shipped canonical generator path so the maintained benchmark and
parity surfaces execute through flash-attention dispatch again on supported requests. The fix must
stay inside the existing generator -> graph -> processor -> kernel chain, preserve the current
quantized `8/4/0/0` runtime contract, and keep the explicit nonflash negative path for unsupported
or out-of-contract requests.

This phase stays inside `src/emel/generator`, maintained generator/parity/bench proof surfaces,
and the related planning/publication artifacts needed to keep the repo truthful. It must not
change Boost.SML state-machine structure, public C API boundaries, or invent a tool-only runtime
path.

</domain>

<decisions>
## Implementation Decisions

### Runtime Restoration
- Restore flash dispatch in the shipped generator hot path by reusing the existing flash-attention
  request helper and kernel dispatch seam already present in `src/emel/generator/detail.hpp`.
- Treat the current manual `compute_attention(...)` path as the explicit nonflash fallback for
  unsupported or out-of-contract cases only, not as the supported canonical runtime.
- Preserve the Phase 22-25 audited `8/4/0/0` quantized runtime contract while changing only the
  attention execution seam.

### Proof And Regression
- Reproduce the regression first with failing tests before changing runtime code, per `AGENTS.md`.
- Restore the maintained proof contract to the earlier flash-attribution truth: supported
  canonical runs must show flash dispatch, and AArch64 canonical runs must show optimized flash
  dispatch with no shared-flash fallback.
- Keep the existing `flash_kv_width_mismatch` generator fixture as the explicit nonflash negative
  surface.

### Publication Guardrails
- Keep the acceptance boundary on `tools/paritychecker`, `tools/bench`, and generator regression
  tests.
- If stored benchmark/docs artifacts need refresh after the runtime fix, follow the existing
  snapshot-update approval rule from `AGENTS.md`.

### the agent's Discretion
- The exact helper split between supported flash dispatch and explicit nonflash fallback can stay
  local to generator/detail helpers as long as the shipped actor chain and public surfaces remain
  unchanged.

</decisions>
