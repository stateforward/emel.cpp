# Phase 51 Plan 01

## Objective

Extend the maintained `embeddings/generator` actor with a truthful in-memory audio lane that runs
the real TE audio tower (`EfficientAT mn20_as`) plus the shared projection head and returns a
normalized `1280`-dimensional embedding.

## Tasks

1. Make the `omniembed` model contract expose the maintained audio preprocessing truth by loading
   `omniembed.audio_encoder_name` and deriving the official EfficientAT `mn20_as` frontend
   contract into repo-owned model metadata, with loader tests proving the derived fields.
2. Extend `src/emel/embeddings/generator/` with an explicit `embed_audio` request path and
   audio-specific guards/actions that validate one bounded mono `float32` PCM memory contract,
   resample the maintained `16 kHz` input to the encoder-native `32 kHz` rate, run EfficientAT
   preprocessing (`preemphasis -> STFT -> mel banks -> log -> normalize`), execute the real
   `audio_encoder.features.*` tower to its pooled `1920`-channel feature vector, apply
   `audio_projection.*`, and publish the normalized shared-space embedding through the existing
   result contract.
3. Add maintained audio tests that synthesize the canonical `pure-tone-440hz` payload in memory
   and verify normalized `1280`-dimensional output plus explicit invalid-payload rejection, then
   run focused tests and full quality gates until the whole repo is green.

## Verification

- `cmake --build build/zig --target emel_tests_bin -j4`
- `./build/zig/emel_tests_bin --no-breaks --source-file='*tests/embeddings/*'`
- `./build/zig/emel_tests_bin --no-breaks --source-file='*tests/model/*'`
- `scripts/quality_gates.sh`

## Done Condition

EMEL accepts the maintained in-memory `pure-tone-440hz` mono PCM payload, preprocesses it
according to the derived EfficientAT `mn20_as` frontend contract, runs the TE audio tower and
audio projection natively in `src/`, returns a normalized `1280`-dimensional embedding, and
rejects malformed audio payloads explicitly while keeping the repo-wide gates green.
