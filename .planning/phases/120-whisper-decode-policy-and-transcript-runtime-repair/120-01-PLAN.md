# Plan 120.1: Whisper Decode Policy And Transcript Runtime Repair

<objective>
Repair the maintained Whisper decoder runtime so `POLICY-01` is source-backed by a speech-owned
decode policy consumed during decoder dispatch, and repair `TOK-02` by removing the decoder's
hardcoded `token:<id>` transcript surface. The maintained publication path must keep transcript
text under `speech/tokenizer/whisper::decode_token_ids`.
</objective>

<rule_constraints>
- `docs/rules/sml.rules.md:3.5`, tasks 2-4: no heap allocation during decoder dispatch; policy
  references and output spans must be caller-owned dispatch-local payload.
- `docs/rules/sml.rules.md:4.8`, task 2: required decoder event fields must be references, not
  nullable pointers.
- `docs/rules/sml.rules.md:5.8`, tasks 2-4: per-dispatch policy, token outputs, and scratch
  buffers stay in event payloads and typed completion propagation, not actor context.
- `docs/rules/sml.rules.md:6.1`, task 3: guards stay pure predicates of event/context.
- `docs/rules/sml.rules.md:6.7`, `6.8`, and `11.1`, tasks 3-4: runtime policy validation and
  output-capacity routing are explicit SML guards/transitions; actions and detail helpers only run
  the already-selected decode algorithm.
- `AGENTS.md` boost.SML actor model, tasks 2-4: no process/defer queues, no self-dispatch, no
  dynamic allocation, and unexpected-event handling remains explicit.
- `AGENTS.md` architecture and composition, tasks 2-5: transition rows remain destination-first;
  Whisper runtime stays under `src/emel/speech/encoder/whisper/**`,
  `src/emel/speech/decoder/whisper/**`, and `src/emel/speech/tokenizer/whisper/**`.
- `AGENTS.md` events outcomes and errors, task 2: public events use noun-like names and failures
  use explicit decoder error codes.
- `AGENTS.md` context rules, tasks 2-4: no request, output pointer, phase, status, or per-dispatch
  policy data is stored in decoder context.
- `AGENTS.md` reference policy, tasks 5-6: closeout claims must be verified against source paths,
  compare output, benchmark output, and domain-boundary checks.
</rule_constraints>
