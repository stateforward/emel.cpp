# Phase 40.1 Summary

Phase 40.1 closed the remaining maintained Bonsai generation gap by adding a real Qwen-family
RoPE domain to `src/emel` instead of leaving EMEL on plain adjacent-pair RoPE. `src/emel/model`
now publishes rope type and rope scaling truth from GGUF metadata, including the Qwen-family
`neox` pair layout and the YaRN inputs needed by the maintained Bonsai slice.

`src/emel/generator/detail.hpp` now derives the runtime rope domain explicitly during prepare,
rejects unsupported domains as `model_invalid`, and applies Qwen-family rotary behavior with the
correct pair layout plus YaRN angle and magnitude scaling. The implementation stays domain-shaped:
no `bonsai` execution family was introduced, and no Bonsai-only runtime branch was added.

Bench and parity tooling now reuse the same shared rope metadata helper, which keeps the EMEL lane
and the reference lane aligned on loaded model truth while preserving lane isolation. With the
runtime domain corrected, the maintained Qwen generation fixture was moved from an append-only
snapshot contract to live reference generation so the existing Qwen anchor remains an honest
maintained comparator rather than a stale stored output.
