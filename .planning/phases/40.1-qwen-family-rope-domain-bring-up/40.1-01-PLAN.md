# Phase 40.1 Plan 01

## Objective

Add an explicit Qwen-family RoPE domain to the shipped EMEL runtime so the maintained Bonsai slice
reaches truthful live-reference generation parity without any Bonsai-specific runtime behavior.

## Tasks

1. Reproduce the remaining first-token Bonsai divergence as a rotary-domain failure instead of a
   formatter or quantized-kernel failure.
2. Publish rope type and rope scaling metadata from GGUF into `model::data`, including the Qwen
   `neox` pair layout and YaRN inputs required by the maintained slice.
3. Port the effective Qwen-family RoPE arithmetic into `src/emel/generator/detail.hpp`, including
   `neox` pairing, YaRN angle scaling, and YaRN magnitude scaling, with explicit runtime-domain
   validation for unsupported combinations.
4. Align parity and bench tooling with the shared rope metadata helper and keep the maintained Qwen
   generation anchor on a truthful live-reference contract.
