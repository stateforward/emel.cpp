## Cross-Phase Integration

**Status:** source-level integration passed.

- Phase `26` anchors the canonical fixture and the GGUF-derived formatter contract used by both
  maintained tools.
- Phase `26.1` adds native `q8_0` runtime support in `src/emel`, which Phase `27` then consumes to
  bring the shipped generator up on the canonical `qwen3` slice.
- Phase `28` reuses that same runtime and formatter contract on the maintained parity surface and
  keeps the prior Llama anchor protected.
- Phase `29` reuses the same canonical slice on compare output, stored benchmark evidence, and docs
  publication instead of inventing a separate Qwen harness.

The only integration debt found is environment-level provenance drift: a local `paritychecker`
build can point at a different fetched reference revision than the pinned bench/docs publication
build. That does not break the source wiring or the milestone requirements, but it should be
normalized so parity and benchmark publication cannot silently diverge in future audits.
